Title: PyBuffer_FillInfo Function Overview
URL: https://docs.python.org/3/c-api/buffer.html
Summary: The `PyBuffer_FillInfo` function handles buffer requests from exporters, allowing exposure of a buffer (`buf`) of a specified size (`len`) with controlled writability. The function fills in the `view` structure based on provided flags, ensuring compliance with read-only constraints. This function has been part of the Stable ABI since version 3.11.
---

intPyBuffer_FillInfo([Py_buffer](https://docs.python.org/3/c-api/buffer.html#c.Py_buffer "Py_buffer")*view, [PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*exporter, void*buf, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")len, intreadonly, intflags)[¶](https://docs.python.org/3/c-api/buffer.html#c.PyBuffer_FillInfo "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable) since version 3.11._
Handle buffer requests for an exporter that wants to expose _buf_ of size _len_ with writability set according to _readonly_. _buf_ is interpreted as a sequence of unsigned bytes.
The _flags_ argument indicates the request type. This function always fills in _view_ as specified by flags, unless _buf_ has been designated as read-only and [`PyBUF_WRITABLE`](https://docs.python.org/3/c-api/buffer.html#c.PyBUF_WRITABLE "PyBUF_WRITABLE") is set in _flags_.
On success, set `view->obj` to a new reference to _exporter_ and return 0. Otherwise, raise [`BufferError`](https://docs.python.org/3/library/exceptions.html#BufferError "BufferError"), set `view->obj` to `NULL` and return `-1`;
If this function is used as part of a [getbufferproc](https://docs.python.org/3/c-api/typeobj.html#buffer-structs), _exporter_ MUST be set to the exporting object and _flags_ must be passed unmodified. Otherwise, _exporter_ MUST be `NULL`.

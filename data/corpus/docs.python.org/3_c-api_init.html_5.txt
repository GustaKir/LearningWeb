Title: Python C API Initialization Flags
URL: https://docs.python.org/3/c-api/init.html
Summary: This document details the `Py_OptimizeFlag` and `Py_QuietFlag`, which are API components maintained for backward compatibility. Users are advised to use the `PyConfig` structure for setting optimization and quiet flags instead. Both flags are deprecated since Python 3.12 and are set via command-line options and environment variables.
---

intPy_OptimizeFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_OptimizeFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.optimization_level`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.optimization_level "PyConfig.optimization_level") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Set by the [`-O`](https://docs.python.org/3/using/cmdline.html#cmdoption-O) option and the [`PYTHONOPTIMIZE`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONOPTIMIZE) environment variable.
Deprecated since version 3.12, will be removed in version 3.14. 

intPy_QuietFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_QuietFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.quiet`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.quiet "PyConfig.quiet") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Don’t display the copyright and version messages even in interactive mode.
Set by the [`-q`](https://docs.python.org/3/using/cmdline.html#cmdoption-q) option.
Added in version 3.2.
Deprecated since version 3.12, will be removed in version 3.14. 

intPy_UnbufferedStdioFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_UnbufferedStdioFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.buffered_stdio`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.buffered_stdio "PyConfig.buffered_stdio") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Force the stdout and stderr streams to be unbuffered.
Set by the [`-u`](https://docs.python.org/3/using/cmdline.html#cmdoption-u) option and the [`PYTHONUNBUFFERED`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONUNBUFFERED) environment variable.
Deprecated since version 3.12, will be removed in version 3.14. 

intPy_VerboseFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_VerboseFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.verbose`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.verbose "PyConfig.verbose") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Print a message each time a module is initialized, showing the place (filename or built-in module) from which it is loaded. If greater or equal to `2`, print a message for each file that is checked for when searching for a module. Also provides information on module cleanup at exit.
Set by the [`-v`](https://docs.python.org/3/using/cmdline.html#cmdoption-v) option and the [`PYTHONVERBOSE`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONVERBOSE) environment variable.
Deprecated since version 3.12, will be removed in version 3.14.

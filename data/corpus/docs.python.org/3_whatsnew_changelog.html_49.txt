Title: Python 3 Changelog Highlights
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This document details notable changes in Python 3, such as improvements to tracebacks in the InteractiveInterpreter and the adjustment of certain constants on Android. It references specific GitHub issues related to these updates.
---

  * [gh-122478](https://github.com/python/cpython/issues/122478): Remove internal frames from tracebacks shown in [`code.InteractiveInterpreter`](https://docs.python.org/3/library/code.html#code.InteractiveInterpreter "code.InteractiveInterpreter") with non-default [`sys.excepthook()`](https://docs.python.org/3/library/sys.html#sys.excepthook "sys.excepthook"). Save correct tracebacks in [`sys.last_traceback`](https://docs.python.org/3/library/sys.html#sys.last_traceback "sys.last_traceback") and update `__traceback__` attribute of [`sys.last_value`](https://docs.python.org/3/library/sys.html#sys.last_value "sys.last_value") and [`sys.last_exc`](https://docs.python.org/3/library/sys.html#sys.last_exc "sys.last_exc").
  * [gh-116622](https://github.com/python/cpython/issues/116622): On Android, the `FICLONE` and `FICLONERANGE` constants are no longer exposed by [`fcntl`](https://docs.python.org/3/library/fcntl.html#module-fcntl "fcntl: The fcntl\(\) and ioctl\(\) system calls. \(Unix\)"), as these ioctls are blocked by SELinux.
  * [gh-82378](https://github.com/python/cpython/issues/82378): Make sure that the new [REPL](https://docs.python.org/3/glossary.html#term-REPL) interprets [`sys.tracebacklimit`](https://docs.python.org/3/library/sys.html#sys.tracebacklimit "sys.tracebacklimit") in the same way that the classic REPL did.
  * [gh-122334](https://github.com/python/cpython/issues/122334): Fix crash when importing [`ssl`](https://docs.python.org/3/library/ssl.html#module-ssl "ssl: TLS/SSL wrapper for socket objects") after the main interpreter restarts.
  * [gh-87320](https://github.com/python/cpython/issues/87320): In [`code.InteractiveInterpreter`](https://docs.python.org/3/library/code.html#code.InteractiveInterpreter "code.InteractiveInterpreter"), handle exceptions caused by calling a non-default [`sys.excepthook()`](https://docs.python.org/3/library/sys.html#sys.excepthook "sys.excepthook"). Before, the exception bubbled up to the caller, ending the [REPL](https://docs.python.org/3/glossary.html#term-REPL).
  * [gh-121650](https://github.com/python/cpython/issues/121650): [`email`](https://docs.python.org/3/library/email.html#module-email "email: Package supporting the parsing, manipulating, and generating email messages.") headers with embedded newlines are now quoted on output. The [`generator`](https://docs.python.org/3/library/email.generator.html#module-email.generator "email.generator: Generate flat text email messages from a message structure.") will now refuse to serialize (write) headers that are unsafely folded or delimited; see [`verify_generated_headers`](https://docs.python.org/3/library/email.policy.html#email.policy.Policy.verify_generated_headers "email.policy.Policy.verify_generated_headers"). (Contributed by Bas Bloemsaat and Petr Viktorin in [gh-121650](https://github.com/python/cpython/issues/121650).)
  * [gh-121723](https://github.com/python/cpython/issues/121723): Make [`logging.config.dictConfig()`](https://docs.python.org/3/library/logging.config.html#logging.config.dictConfig "logging.config.dictConfig") accept any object implementing the Queue public API. See the [queue configuration](https://docs.python.org/3/library/logging.config.html#configure-queue) section for details. Patch by Bénédikt Tran.
  * [gh-122081](https://github.com/python/cpython/issues/122081): Fix a crash in the `decimal.IEEEContext()` optional function available via the `EXTRA_FUNCTIONALITY` configuration flag.
  * [gh-121804](https://github.com/python/cpython/issues/121804): Correctly show error locations, when [`SyntaxError`](https://docs.python.org/3/library/exceptions.html#SyntaxError "SyntaxError") raised in new repl. Patch by Sergey B Kirpichev.
  * [gh-121151](https://github.com/python/cpython/issues/121151): Fix wrapping of long usage text of arguments inside a mutually exclusive group in [`argparse`](https://docs.python.org/3/library/argparse.html#module-argparse "argparse: Command-line option and argument parsing library.").
  * [gh-108172](https://github.com/python/cpython/issues/108172): `webbrowser` honors OS preferred browser on Linux when its desktop entry name contains the text of a known browser name.
  * [gh-109109](https://github.com/python/cpython/issues/109109): You can now get the raw TLS certificate chains from TLS connections via [`ssl.SSLSocket.get_verified_chain()`](https://docs.python.org/3/library/ssl.html#ssl.SSLSocket.get_verified_chain "ssl.SSLSocket.get_verified_chain") and [`ssl.SSLSocket.get_unverified_chain()`](https://docs.python.org/3/library/ssl.html#ssl.SSLSocket.get_unverified_chain "ssl.SSLSocket.get_unverified_chain") methods.
Contributed by Mateusz Nowak.

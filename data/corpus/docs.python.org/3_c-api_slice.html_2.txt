Title: Python C API: Slice Functions Updates
URL: https://docs.python.org/3/c-api/slice.html
Summary: This documentation chunk outlines changes to slice-related functions in the Python C API, specifically noting the evolution of `_slice_` parameter types, the implementation details of `PySlice_GetIndicesEx()`, and the deprecation of this function under certain `Py_LIMITED_API` settings.

```

Changed in version 3.2: The parameter type for the _slice_ parameter was `PySliceObject*` before.
Changed in version 3.6.1: If `Py_LIMITED_API` is not set or set to the value between `0x03050400` and `0x03060000` (not including) or `0x03060100` or higher `PySlice_GetIndicesEx()` is implemented as a macro using `PySlice_Unpack()` and `PySlice_AdjustIndices()`. Arguments _start_ , _stop_ and _step_ are evaluated more than once.
Deprecated since version 3.6.1: If `Py_LIMITED_API` is set to the value less than `0x03050400` or between `0x03060000` and `0x03060100` (not including) `PySlice_GetIndicesEx()` is a deprecated function. 

intPySlice_Unpack([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*slice, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")*start, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")*stop, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")*step)[¶](https://docs.python.org/3/c-api/slice.html#c.PySlice_Unpack "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable) since version 3.7._
Extract the start, stop and step data members from a slice object as C integers. Silently reduce values larger than `PY_SSIZE_T_MAX` to `PY_SSIZE_T_MAX`, silently boost the start and stop values less than `PY_SSIZE_T_MIN` to `PY_SSIZE_T_MIN`, and silently boost the step values less than `-PY_SSIZE_T_MAX` to `-PY_SSIZE_T_MAX`.
Return `-1` with an exception set on error, `0` on success.
Added in version 3.6.1. 

[Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")PySlice_AdjustIndices([Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")length, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")*start, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")*stop, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")step)[¶](https://docs.python.org/3/c-api/slice.html#c.PySlice_AdjustIndices "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable) since version 3.7._
Adjust start/end slice indices assuming a sequence of the specified length. Out of bounds indices are clipped in a manner consistent with the handling of normal slices.
Return the length of the slice. Always successful. Doesn’t call Python code.
Added in version 3.6.1.

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*Py_Ellipsis[¶](https://docs.python.org/3/c-api/slice.html#c.Py_Ellipsis "Link to this definition")
    
The Python `Ellipsis` object. This object has no methods. Like [`Py_None`](https://docs.python.org/3/c-api/none.html#c.Py_None "Py_None"), it is an [immortal](https://docs.python.org/3/glossary.html#term-immortal) singleton object.
Changed in version 3.12: [`Py_Ellipsis`](https://docs.python.org/3/c-api/slice.html#c.Py_Ellipsis "Py_Ellipsis") is immortal.

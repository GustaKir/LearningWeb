Title: PyStructSequence_SetItem Function Documentation
URL: https://docs.python.org/3/c-api/tuple.html
Summary: The `PyStructSequence_SetItem` function sets the field at a specified index of a struct sequence to a given value. This function performs bounds checking in debug mode and 'steals' a reference to the value passed in.
---

voidPyStructSequence_SetItem([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*p, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")pos, [PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*o)[¶](https://docs.python.org/3/c-api/tuple.html#c.PyStructSequence_SetItem "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Sets the field at index _pos_ of the struct sequence _p_ to value _o_. Like [`PyTuple_SET_ITEM()`](https://docs.python.org/3/c-api/tuple.html#c.PyTuple_SET_ITEM "PyTuple_SET_ITEM"), this should only be used to fill in brand new instances.
Bounds checking is performed as an assertion if Python is built in [debug mode](https://docs.python.org/3/using/configure.html#debug-build) or [`with assertions`](https://docs.python.org/3/using/configure.html#cmdoption-with-assertions).
Note
This function “steals” a reference to _o_.

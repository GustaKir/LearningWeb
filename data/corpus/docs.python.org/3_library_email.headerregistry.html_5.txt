_class_ email.headerregistry.ParameterizedMIMEHeader[¶](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.ParameterizedMIMEHeader "Link to this definition")
    
MIME headers all start with the prefix ‘Content-’. Each specific header has a certain value, described under the class for that header. Some can also take a list of supplemental parameters, which have a common format. This class serves as a base for all the MIME headers that take parameters. 

params[¶](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.ParameterizedMIMEHeader.params "Link to this definition")
    
A dictionary mapping parameter names to parameter values.

_class_ email.headerregistry.ContentDispositionHeader[¶](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.ContentDispositionHeader "Link to this definition")
    
A [`ParameterizedMIMEHeader`](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.ParameterizedMIMEHeader "email.headerregistry.ParameterizedMIMEHeader") class that handles the _Content-Disposition_ header. 

content_disposition[¶](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.ContentDispositionHeader.content_disposition "Link to this definition")
    
`inline` and `attachment` are the only valid values in common use. 

_class_ email.headerregistry.ContentTransferEncoding[¶](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.ContentTransferEncoding "Link to this definition")
    
Handles the _Content-Transfer-Encoding_ header. 

cte[¶](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.ContentTransferEncoding.cte "Link to this definition")
    
Valid values are `7bit`, `8bit`, `base64`, and `quoted-printable`. See [**RFC 2045**](https://datatracker.ietf.org/doc/html/rfc2045.html) for more information. 

_class_ email.headerregistry.HeaderRegistry(_base_class =BaseHeader_, _default_class =UnstructuredHeader_, _use_default_map =True_)[¶](https://docs.python.org/3/library/email.headerregistry.html#email.headerregistry.HeaderRegistry "Link to this definition")
    
This is the factory used by [`EmailPolicy`](https://docs.python.org/3/library/email.policy.html#email.policy.EmailPolicy "email.policy.EmailPolicy") by default. `HeaderRegistry` builds the class used to create a header instance dynamically, using _base_class_ and a specialized class retrieved from a registry that it holds. When a given header name does not appear in the registry, the class specified by _default_class_ is used as the specialized class. When _use_default_map_ is `True` (the default), the standard mapping of header names to classes is copied in to the registry during initialization. _base_class_ is always the last class in the generated class’s [`__bases__`](https://docs.python.org/3/reference/datamodel.html#type.__bases__ "type.__bases__") list.
The default mappings are:

> UniqueUnstructuredHeader

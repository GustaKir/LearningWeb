Title: Library Updates in Python
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This section outlines recent fixes and changes in the Python library, including corrections in error messages, crash fixes for `_interpreters.run_string`, and the introduction of a new environment variable for subprocess management.
---

### Library[¶](https://docs.python.org/3/whatsnew/changelog.html#id3 "Link to this heading")
  * [gh-132174](https://github.com/python/cpython/issues/132174): Fix function name in error message of `_interpreters.run_string`.
  * [gh-132171](https://github.com/python/cpython/issues/132171): Fix crash of `_interpreters.run_string` on string subclasses.
  * [gh-129204](https://github.com/python/cpython/issues/129204): Introduce new `_PYTHON_SUBPROCESS_USE_POSIX_SPAWN` environment variable knob in [`subprocess`](https://docs.python.org/3/library/subprocess.html#module-subprocess "subprocess: Subprocess management.") to control the use of [`os.posix_spawn()`](https://docs.python.org/3/library/os.html#os.posix_spawn "os.posix_spawn").
  * [gh-132159](https://github.com/python/cpython/issues/132159): Do not shadow user arguments in generated `__new__()` by decorator [`warnings.deprecated`](https://docs.python.org/3/library/warnings.html#warnings.deprecated "warnings.deprecated"). Patch by Xuehai Pan.
  * [gh-132075](https://github.com/python/cpython/issues/132075): Fix possible use of [`socket`](https://docs.python.org/3/library/socket.html#module-socket "socket: Low-level networking interface.") address structures with uninitialized members. Now all structure members are initialized with zeroes by default.
  * [gh-132002](https://github.com/python/cpython/issues/132002): Fix crash when deallocating [`contextvars.ContextVar`](https://docs.python.org/3/library/contextvars.html#contextvars.ContextVar "contextvars.ContextVar") with weird unahashable string names.
  * [gh-131668](https://github.com/python/cpython/issues/131668): [`socket`](https://docs.python.org/3/library/socket.html#module-socket "socket: Low-level networking interface."): Fix code parsing AF_BLUETOOTH socket addresses.
  * [gh-131492](https://github.com/python/cpython/issues/131492): Fix a resource leak when constructing a [`gzip.GzipFile`](https://docs.python.org/3/library/gzip.html#gzip.GzipFile "gzip.GzipFile") with a filename fails, for example when passing an invalid `compresslevel`.
  * [gh-131325](https://github.com/python/cpython/issues/131325): Fix sendfile fallback implementation to drain data after writing to transport in [`asyncio`](https://docs.python.org/3/library/asyncio.html#module-asyncio "asyncio: Asynchronous I/O.").
  * [gh-129843](https://github.com/python/cpython/issues/129843): Fix incorrect argument passing in [`warnings.warn_explicit()`](https://docs.python.org/3/library/warnings.html#warnings.warn_explicit "warnings.warn_explicit").
  * [gh-131204](https://github.com/python/cpython/issues/131204): Use monospace font from System Font Stack for cross-platform support in [`difflib.HtmlDiff`](https://docs.python.org/3/library/difflib.html#difflib.HtmlDiff "difflib.HtmlDiff").
  * [gh-130940](https://github.com/python/cpython/issues/130940): The `PyConfig.use_system_logger` attribute, introduced in Python 3.13.2, has been removed. The introduction of this attribute inadvertently introduced an ABI breakage on macOS and iOS. The use of the system logger is now enabled by default on iOS, and disabled by default on macOS.
  * [gh-131045](https://github.com/python/cpython/issues/131045): Fix issue with `__contains__`, values, and pseudo-members for [`enum.Flag`](https://docs.python.org/3/library/enum.html#enum.Flag "enum.Flag").
  * [gh-130959](https://github.com/python/cpython/issues/130959): Fix pure-Python implementation of [`datetime.time.fromisoformat()`](https://docs.python.org/3/library/datetime.html#datetime.time.fromisoformat "datetime.time.fromisoformat") to reject times with spaces in fractional part (for example, `12:34:56.400 +02:00`), matching the C implementation. Patch by Michał Gorny.
  * [gh-130637](https://github.com/python/cpython/issues/130637): Add validation for numeric response data in poplib.POP3.stat() method
  * [gh-130461](https://github.com/python/cpython/issues/130461): Remove `.. index::` directives from the [`uuid`](https://docs.python.org/3/library/uuid.html#module-uuid "uuid: UUID objects \(universally unique identifiers\) according to RFC 4122") module documentation. These directives previously created entries in the general index for [`getnode()`](https://docs.python.org/3/library/uuid.html#uuid.getnode "uuid.getnode") as well as the [`uuid1()`](https://docs.python.org/3/library/uuid.html#uuid.uuid1 "uuid.uuid1"), [`uuid3()`](https://docs.python.org/3/library/uuid.html#uuid.uuid3 "uuid.uuid3"), [`uuid4()`](https://docs.python.org/3/library/uuid.html#uuid.uuid4 "uuid.uuid4"), and [`uuid5()`](https://docs.python.org/3/library/uuid.html#uuid.uuid5 "uuid.uuid5") constructor functions.
  * [gh-130379](https://github.com/python/cpython/issues/130379): The zipapp module now calculates the list of files to be added to the archive before creating the archive. This avoids accidentally including the target when it is being created in the source directory.

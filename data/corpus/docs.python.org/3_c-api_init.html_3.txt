Title: Python C API: Initialization Configuration
URL: https://docs.python.org/3/c-api/init.html
Summary: This section discusses the deprecated `Py_FrozenFlag` and `Py_HashRandomizationFlag` APIs, which are retained for backward compatibility. It advises using the `PyConfig.pathconfig_warnings` and `PyConfig.hash_seed` for configuration instead. The `Py_FrozenFlag` suppresses error messages related to module search paths and will be removed in version 3.14, while `Py_HashRandomizationFlag` is also marked for deprecated use.
---

intPy_FrozenFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_FrozenFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.pathconfig_warnings`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.pathconfig_warnings "PyConfig.pathconfig_warnings") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Suppress error messages when calculating the module search path in [`Py_GetPath()`](https://docs.python.org/3/c-api/init.html#c.Py_GetPath "Py_GetPath").
Private flag used by `_freeze_module` and `frozenmain` programs.
Deprecated since version 3.12, will be removed in version 3.14. 

intPy_HashRandomizationFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_HashRandomizationFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.hash_seed`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.hash_seed "PyConfig.hash_seed") and [`PyConfig.use_hash_seed`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.use_hash_seed "PyConfig.use_hash_seed") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Set to `1` if the [`PYTHONHASHSEED`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONHASHSEED) environment variable is set to a non-empty string.
If the flag is non-zero, read the [`PYTHONHASHSEED`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONHASHSEED) environment variable to initialize the secret hash seed.
Deprecated since version 3.12, will be removed in version 3.14. 

intPy_IgnoreEnvironmentFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_IgnoreEnvironmentFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.use_environment`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.use_environment "PyConfig.use_environment") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Ignore all `PYTHON*` environment variables, e.g. [`PYTHONPATH`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONPATH) and [`PYTHONHOME`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONHOME), that might be set.
Set by the [`-E`](https://docs.python.org/3/using/cmdline.html#cmdoption-E) and [`-I`](https://docs.python.org/3/using/cmdline.html#cmdoption-I) options.
Deprecated since version 3.12, will be removed in version 3.14. 

intPy_InspectFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_InspectFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.inspect`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.inspect "PyConfig.inspect") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
When a script is passed as first argument or the [`-c`](https://docs.python.org/3/using/cmdline.html#cmdoption-c) option is used, enter interactive mode after executing the script or the command, even when [`sys.stdin`](https://docs.python.org/3/library/sys.html#sys.stdin "sys.stdin") does not appear to be a terminal.
Set by the [`-i`](https://docs.python.org/3/using/cmdline.html#cmdoption-i) option and the [`PYTHONINSPECT`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONINSPECT) environment variable.
Deprecated since version 3.12, will be removed in version 3.14. 

intPy_InteractiveFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_InteractiveFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.interactive`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.interactive "PyConfig.interactive") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Set by the [`-i`](https://docs.python.org/3/using/cmdline.html#cmdoption-i) option.
Deprecated since version 3.12. 

intPy_IsolatedFlag[¶](https://docs.python.org/3/c-api/init.html#c.Py_IsolatedFlag "Link to this definition")
    
This API is kept for backward compatibility: setting [`PyConfig.isolated`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.isolated "PyConfig.isolated") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Run Python in isolated mode. In isolated mode [`sys.path`](https://docs.python.org/3/library/sys.html#sys.path "sys.path") contains neither the script’s directory nor the user’s site-packages directory.
Set by the [`-I`](https://docs.python.org/3/using/cmdline.html#cmdoption-I) option.
Added in version 3.4.
Deprecated since version 3.12, will be removed in version 3.14.

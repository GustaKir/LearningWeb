Title: Field Definitions in Python C API Type Objects
URL: https://docs.python.org/3/c-api/typeobj.html
Summary: This section describes the requirements for the string representation of objects in Python's C API, emphasizing that it should return a friendly string or Unicode object for use in functions like print(). It mentions inheritance of this field by subtypes and the default use of PyObject_Repr() if not set. Additionally, it introduces an optional pointer for the get-attribute function, outlining its signature.
---

```

The function must return a string or a Unicode object. It should be a “friendly” string representation of the object, as this is the representation that will be used, among other things, by the [`print()`](https://docs.python.org/3/library/functions.html#print "print") function.
**Inheritance:**
This field is inherited by subtypes.
**Default:**
When this field is not set, [`PyObject_Repr()`](https://docs.python.org/3/c-api/object.html#c.PyObject_Repr "PyObject_Repr") is called to return a string representation. 

[getattrofunc](https://docs.python.org/3/c-api/typeobj.html#c.getattrofunc "getattrofunc")[PyTypeObject](https://docs.python.org/3/c-api/type.html#c.PyTypeObject "PyTypeObject").tp_getattro[¶](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattro "Link to this definition")
    
An optional pointer to the get-attribute function.
The signature is the same as for [`PyObject_GetAttr()`](https://docs.python.org/3/c-api/object.html#c.PyObject_GetAttr "PyObject_GetAttr"):
```
PyObject*tp_getattro(PyObject*self,PyObject*attr);

```

It is usually convenient to set this field to [`PyObject_GenericGetAttr()`](https://docs.python.org/3/c-api/object.html#c.PyObject_GenericGetAttr "PyObject_GenericGetAttr"), which implements the normal way of looking for object attributes.
**Inheritance:**
Group: [`tp_getattr`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattr "PyTypeObject.tp_getattr"), [`tp_getattro`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattro "PyTypeObject.tp_getattro")
This field is inherited by subtypes together with [`tp_getattr`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattr "PyTypeObject.tp_getattr"): a subtype inherits both [`tp_getattr`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattr "PyTypeObject.tp_getattr") and [`tp_getattro`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattro "PyTypeObject.tp_getattro") from its base type when the subtype’s [`tp_getattr`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattr "PyTypeObject.tp_getattr") and [`tp_getattro`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_getattro "PyTypeObject.tp_getattro") are both `NULL`.
**Default:**
[`PyBaseObject_Type`](https://docs.python.org/3/c-api/structures.html#c.PyBaseObject_Type "PyBaseObject_Type") uses [`PyObject_GenericGetAttr()`](https://docs.python.org/3/c-api/object.html#c.PyObject_GenericGetAttr "PyObject_GenericGetAttr").

An optional pointer to the function for setting and deleting attributes.
```

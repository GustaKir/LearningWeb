Title: Python 3.7.0 Final Release Notes
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This section details several bug fixes and changes in Python 3.7.0, including the ability for embedding applications to reset warning options before initializing Python, documentation updates regarding the state of multi-phase initialized modules, and changes in the caching behavior of certain Unicode functions.

  * [bpo-30863](https://bugs.python.org/issue?@action=redirect&bpo=30863): [`PyUnicode_AsWideChar()`](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_AsWideChar "PyUnicode_AsWideChar") and [`PyUnicode_AsWideCharString()`](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_AsWideCharString "PyUnicode_AsWideCharString") no longer cache the `wchar_t*` representation of string objects.


## Python 3.7.0 final[¶](https://docs.python.org/3/whatsnew/changelog.html#python-3-7-0-final "Link to this heading")
_Release date: 2018-06-27_
### Library[¶](https://docs.python.org/3/whatsnew/changelog.html#id484 "Link to this heading")
  * [bpo-33851](https://bugs.python.org/issue?@action=redirect&bpo=33851): Fix [`ast.get_docstring()`](https://docs.python.org/3/library/ast.html#ast.get_docstring "ast.get_docstring") for a node that lacks a docstring.


### C API[¶](https://docs.python.org/3/whatsnew/changelog.html#id485 "Link to this heading")
  * [bpo-33932](https://bugs.python.org/issue?@action=redirect&bpo=33932): Calling Py_Initialize() twice does nothing, instead of failing with a fatal error: restore the Python 3.6 behaviour.


## Python 3.7.0 release candidate 1[¶](https://docs.python.org/3/whatsnew/changelog.html#python-3-7-0-release-candidate-1 "Link to this heading")
_Release date: 2018-06-12_
### Core and Builtins[¶](https://docs.python.org/3/whatsnew/changelog.html#id486 "Link to this heading")
  * [bpo-33803](https://bugs.python.org/issue?@action=redirect&bpo=33803): Fix a crash in hamt.c caused by enabling GC tracking for an object that hadn’t all of its fields set to NULL.
  * [bpo-33706](https://bugs.python.org/issue?@action=redirect&bpo=33706): Fix a crash in Python initialization when parsing the command line options. Thanks Christoph Gohlke for the bug report and the fix!
  * [bpo-30654](https://bugs.python.org/issue?@action=redirect&bpo=30654): Fixed reset of the SIGINT handler to SIG_DFL on interpreter shutdown even when there was a custom handler set previously. Patch by Philipp Kerling.
  * [bpo-31849](https://bugs.python.org/issue?@action=redirect&bpo=31849): Fix signed/unsigned comparison warning in pyhash.c.

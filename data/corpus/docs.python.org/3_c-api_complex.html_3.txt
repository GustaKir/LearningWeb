Title: PyComplex_AsCComplex Function
URL: https://docs.python.org/3/c-api/complex.html
Summary: The PyComplex_AsCComplex function extracts the Py_complex value from a given Python complex number object. If the object is not a complex number but implements the __complex__() method, it will invoke this method for conversion. Fallbacks to __float__() and __index__() are applied if necessary.
---

[Py_complex](https://docs.python.org/3/c-api/complex.html#c.Py_complex "Py_complex")PyComplex_AsCComplex([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*op)[¶](https://docs.python.org/3/c-api/complex.html#c.PyComplex_AsCComplex "Link to this definition")
    
Return the [`Py_complex`](https://docs.python.org/3/c-api/complex.html#c.Py_complex "Py_complex") value of the complex number _op_.
If _op_ is not a Python complex number object but has a [`__complex__()`](https://docs.python.org/3/reference/datamodel.html#object.__complex__ "object.__complex__") method, this method will first be called to convert _op_ to a Python complex number object. If `__complex__()` is not defined then it falls back to [`__float__()`](https://docs.python.org/3/reference/datamodel.html#object.__float__ "object.__float__"). If `__float__()` is not defined then it falls back to [`__index__()`](https://docs.python.org/3/reference/datamodel.html#object.__index__ "object.__index__").
Upon failure, this method returns [`Py_complex`](https://docs.python.org/3/c-api/complex.html#c.Py_complex "Py_complex") with [`real`](https://docs.python.org/3/c-api/complex.html#c.Py_complex.real "Py_complex.real") set to `-1.0` and with an exception set, so one should call [`PyErr_Occurred()`](https://docs.python.org/3/c-api/exceptions.html#c.PyErr_Occurred "PyErr_Occurred") to check for errors.
Changed in version 3.8: Use [`__index__()`](https://docs.python.org/3/reference/datamodel.html#object.__index__ "object.__index__") if available.

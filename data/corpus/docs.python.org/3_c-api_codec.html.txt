# Codec registry and support functions[¶](https://docs.python.org/3/c-api/codec.html#codec-registry-and-support-functions "Link to this heading") 

intPyCodec_Register([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*search_function)[¶](https://docs.python.org/3/c-api/codec.html#c.PyCodec_Register "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Register a new codec search function.
As side effect, this tries to load the `encodings` package, if not yet done, to make sure that it is always first in the list of search functions. 

intPyCodec_Unregister([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*search_function)[¶](https://docs.python.org/3/c-api/codec.html#c.PyCodec_Unregister "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable) since version 3.10._
Unregister a codec search function and clear the registry’s cache. If the search function is not registered, do nothing. Return 0 on success. Raise an exception and return -1 on error.
Added in version 3.10. 

intPyCodec_KnownEncoding(constchar*encoding)[¶](https://docs.python.org/3/c-api/codec.html#c.PyCodec_KnownEncoding "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Return `1` or `0` depending on whether there is a registered codec for the given _encoding_. This function always succeeds. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyCodec_Encode([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*object, constchar*encoding, constchar*errors)[¶](https://docs.python.org/3/c-api/codec.html#c.PyCodec_Encode "Link to this definition")
    _Return value: New reference.__Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Generic codec based encoding API.
_object_ is passed through the encoder function found for the given _encoding_ using the error handling method defined by _errors_. _errors_ may be `NULL` to use the default method defined for the codec. Raises a [`LookupError`](https://docs.python.org/3/library/exceptions.html#LookupError "LookupError") if no encoder can be found.

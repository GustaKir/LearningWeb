Title: Py_SetProgramName Function
URL: https://docs.python.org/3/c-api/init.html
Summary: The `Py_SetProgramName` function, part of the Stable ABI, is used for backward compatibility to set the program name for the Python interpreter. It should be called before `Py_Initialize()`, specifying the `argv[0]` argument as a wide character string. However, for newer applications, `PyConfig.program_name` is recommended for setting the program name.
---

voidPy_SetProgramName(constwchar_t*name)[¶](https://docs.python.org/3/c-api/init.html#c.Py_SetProgramName "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
This API is kept for backward compatibility: setting [`PyConfig.program_name`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.program_name "PyConfig.program_name") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
This function should be called before [`Py_Initialize()`](https://docs.python.org/3/c-api/init.html#c.Py_Initialize "Py_Initialize") is called for the first time, if it is called at all. It tells the interpreter the value of the `argv[0]` argument to the `main()` function of the program (converted to wide characters). This is used by [`Py_GetPath()`](https://docs.python.org/3/c-api/init.html#c.Py_GetPath "Py_GetPath") and some other functions below to find the Python run-time libraries relative to the interpreter executable. The default value is `'python'`. The argument should point to a zero-terminated wide character string in static storage whose contents will not change for the duration of the program’s execution. No code in the Python interpreter will change the contents of this storage.
Use [`Py_DecodeLocale()`](https://docs.python.org/3/c-api/sys.html#c.Py_DecodeLocale "Py_DecodeLocale") to decode a bytes string to get a wchar_t* string.
Deprecated since version 3.11. 

wchar_t*Py_GetProgramName()[¶](https://docs.python.org/3/c-api/init.html#c.Py_GetProgramName "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Return the program name set with [`PyConfig.program_name`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.program_name "PyConfig.program_name"), or the default. The returned string points into static storage; the caller should not modify its value.
This function should not be called before [`Py_Initialize()`](https://docs.python.org/3/c-api/init.html#c.Py_Initialize "Py_Initialize"), otherwise it returns `NULL`.
Changed in version 3.10: It now returns `NULL` if called before [`Py_Initialize()`](https://docs.python.org/3/c-api/init.html#c.Py_Initialize "Py_Initialize").
Deprecated since version 3.13, will be removed in version 3.15: Get [`sys.executable`](https://docs.python.org/3/library/sys.html#sys.executable "sys.executable") instead. 

wchar_t*Py_GetPrefix()[¶](https://docs.python.org/3/c-api/init.html#c.Py_GetPrefix "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Return the _prefix_ for installed platform-independent files. This is derived through a number of complicated rules from the program name set with [`PyConfig.program_name`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.program_name "PyConfig.program_name") and some environment variables; for example, if the program name is `'/usr/local/bin/python'`, the prefix is `'/usr/local'`. The returned string points into static storage; the caller should not modify its value. This corresponds to the **prefix** variable in the top-level `Makefile` and the [`--prefix`](https://docs.python.org/3/using/configure.html#cmdoption-prefix) argument to the **configure** script at build time. The value is available to Python code as `sys.base_prefix`. It is only useful on Unix. See also the next function.
This function should not be called before [`Py_Initialize()`](https://docs.python.org/3/c-api/init.html#c.Py_Initialize "Py_Initialize"), otherwise it returns `NULL`.
Changed in version 3.10: It now returns `NULL` if called before [`Py_Initialize()`](https://docs.python.org/3/c-api/init.html#c.Py_Initialize "Py_Initialize").
Deprecated since version 3.13, will be removed in version 3.15: Get [`sys.base_prefix`](https://docs.python.org/3/library/sys.html#sys.base_prefix "sys.base_prefix") instead, or [`sys.prefix`](https://docs.python.org/3/library/sys.html#sys.prefix "sys.prefix") if [virtual environments](https://docs.python.org/3/library/venv.html#venv-def) need to be handled.

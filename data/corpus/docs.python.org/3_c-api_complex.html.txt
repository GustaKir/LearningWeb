# Complex Number Objects[¶](https://docs.python.org/3/c-api/complex.html#complex-number-objects "Link to this heading")
Python’s complex number objects are implemented as two distinct types when viewed from the C API: one is the Python object exposed to Python programs, and the other is a C structure which represents the actual complex number value. The API provides functions for working with both.
## Complex Numbers as C Structures[¶](https://docs.python.org/3/c-api/complex.html#complex-numbers-as-c-structures "Link to this heading")
Note that the functions which accept these structures as parameters and return them as results do so _by value_ rather than dereferencing them through pointers. This is consistent throughout the API. 

typePy_complex[¶](https://docs.python.org/3/c-api/complex.html#c.Py_complex "Link to this definition")
    
The C structure which corresponds to the value portion of a Python complex number object. Most of the functions for dealing with complex number objects use structures of this type as input or output values, as appropriate. 

doublereal[¶](https://docs.python.org/3/c-api/complex.html#c.Py_complex.real "Link to this definition")


doubleimag[¶](https://docs.python.org/3/c-api/complex.html#c.Py_complex.imag "Link to this definition")

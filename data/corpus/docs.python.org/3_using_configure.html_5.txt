Title: Configure Options for libmpdec and liblzma
URL: https://docs.python.org/3/using/configure.html
Summary: This section describes the C compiler and linker flags for `libmpdec` and `liblzma`, which are used by the `decimal` and `lzma` modules respectively. It notes that these environment variables are ineffective unless `--with-system-libmpdec` is specified.
---

LIBMPDEC_LIBS[¶](https://docs.python.org/3/using/configure.html#cmdoption-arg-LIBMPDEC_LIBS "Link to this definition")
    
C compiler and linker flags for `libmpdec`, used by [`decimal`](https://docs.python.org/3/library/decimal.html#module-decimal "decimal: Implementation of the General Decimal Arithmetic Specification.") module, overriding `pkg-config`.
Note
These environment variables have no effect unless [`--with-system-libmpdec`](https://docs.python.org/3/using/configure.html#cmdoption-with-system-libmpdec) is specified.

### 3.3.5. WebAssembly Options[¶](https://docs.python.org/3/using/configure.html#webassembly-options "Link to this heading") 

--with-emscripten-target=[browser|node][¶](https://docs.python.org/3/using/configure.html#cmdoption-with-emscripten-target "Link to this definition")
    
Set build flavor for `wasm32-emscripten`.
  * `browser` (default): preload minimal stdlib, default MEMFS.
  * `node`: NODERAWFS and pthread support.


Added in version 3.11. 

--enable-wasm-dynamic-linking[¶](https://docs.python.org/3/using/configure.html#cmdoption-enable-wasm-dynamic-linking "Link to this definition")
    
Turn on dynamic linking support for WASM.
Dynamic linking enables `dlopen`. File size of the executable increases due to limited dead code elimination and additional features.
Added in version 3.11. 

--enable-wasm-pthreads[¶](https://docs.python.org/3/using/configure.html#cmdoption-enable-wasm-pthreads "Link to this definition")
    
Turn on pthreads support for WASM.
Added in version 3.11.

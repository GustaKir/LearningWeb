Title: What's New in Python 3.3 - Enhancements and Contributions
URL: https://docs.python.org/3/whatsnew/3.3.html
Summary: Python 3.3 introduces updates to `ctypes` and `os.symlink()`, allowing cross-platform support. Additionally, the `pdb` module now features tab-completion for command arguments, enhancing user experience. Contributions from developers are noted in specific issue tracking references.

### pdb[¶](https://docs.python.org/3/whatsnew/3.3.html#pdb "Link to this heading")
Tab-completion is now available not only for command names, but also their arguments. For example, for the `break` command, function and file names are completed.
(Contributed by Georg Brandl in [bpo-14210](https://bugs.python.org/issue?@action=redirect&bpo=14210))
### pickle[¶](https://docs.python.org/3/whatsnew/3.3.html#pickle "Link to this heading")
[`pickle.Pickler`](https://docs.python.org/3/library/pickle.html#pickle.Pickler "pickle.Pickler") objects now have an optional [`dispatch_table`](https://docs.python.org/3/library/pickle.html#pickle.Pickler.dispatch_table "pickle.Pickler.dispatch_table") attribute allowing per-pickler reduction functions to be set.
(Contributed by Richard Oudkerk in [bpo-14166](https://bugs.python.org/issue?@action=redirect&bpo=14166).)
### pydoc[¶](https://docs.python.org/3/whatsnew/3.3.html#pydoc "Link to this heading")
The Tk GUI and the `serve()` function have been removed from the [`pydoc`](https://docs.python.org/3/library/pydoc.html#module-pydoc "pydoc: Documentation generator and online help system.") module: `pydoc -g` and `serve()` have been deprecated in Python 3.2.
### re[¶](https://docs.python.org/3/whatsnew/3.3.html#re "Link to this heading")
[`str`](https://docs.python.org/3/library/stdtypes.html#str "str") regular expressions now support `\u` and `\U` escapes.
(Contributed by Serhiy Storchaka in [bpo-3665](https://bugs.python.org/issue?@action=redirect&bpo=3665).)
### sched[¶](https://docs.python.org/3/whatsnew/3.3.html#sched "Link to this heading")
  * [`run()`](https://docs.python.org/3/library/sched.html#sched.scheduler.run "sched.scheduler.run") now accepts a _blocking_ parameter which when set to false makes the method execute the scheduled events due to expire soonest (if any) and then return immediately. This is useful in case you want to use the [`scheduler`](https://docs.python.org/3/library/sched.html#sched.scheduler "sched.scheduler") in non-blocking applications. (Contributed by Giampaolo Rodolà in [bpo-13449](https://bugs.python.org/issue?@action=redirect&bpo=13449).)
  * [`scheduler`](https://docs.python.org/3/library/sched.html#sched.scheduler "sched.scheduler") class can now be safely used in multi-threaded environments. (Contributed by Josiah Carlson and Giampaolo Rodolà in [bpo-8684](https://bugs.python.org/issue?@action=redirect&bpo=8684).)
  * _timefunc_ and _delayfunct_ parameters of [`scheduler`](https://docs.python.org/3/library/sched.html#sched.scheduler "sched.scheduler") class constructor are now optional and defaults to [`time.time()`](https://docs.python.org/3/library/time.html#time.time "time.time") and [`time.sleep()`](https://docs.python.org/3/library/time.html#time.sleep "time.sleep") respectively. (Contributed by Chris Clark in [bpo-13245](https://bugs.python.org/issue?@action=redirect&bpo=13245).)
  * [`enter()`](https://docs.python.org/3/library/sched.html#sched.scheduler.enter "sched.scheduler.enter") and [`enterabs()`](https://docs.python.org/3/library/sched.html#sched.scheduler.enterabs "sched.scheduler.enterabs") _argument_ parameter is now optional. (Contributed by Chris Clark in [bpo-13245](https://bugs.python.org/issue?@action=redirect&bpo=13245).)
  * [`enter()`](https://docs.python.org/3/library/sched.html#sched.scheduler.enter "sched.scheduler.enter") and [`enterabs()`](https://docs.python.org/3/library/sched.html#sched.scheduler.enterabs "sched.scheduler.enterabs") now accept a _kwargs_ parameter. (Contributed by Chris Clark in [bpo-13245](https://bugs.python.org/issue?@action=redirect&bpo=13245).)

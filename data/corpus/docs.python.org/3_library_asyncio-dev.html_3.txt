Title: Handling Task Exceptions in asyncio
URL: https://docs.python.org/3/library/asyncio-dev.html
Summary: This section explains how to handle exceptions in asyncio tasks using a traceback to diagnose issues. It demonstrates enabling debug mode to obtain detailed information about task exceptions, including their origins and the errors raised during execution.
---

```
Task exception was never retrieved
future: <Task finished coro=<bug() done, defined at test.py:3>
 exception=Exception('not consumed')>
Traceback (most recent call last):
 File "test.py", line 4, in bug
  raise Exception("not consumed")
Exception: not consumed

```

[Enable the debug mode](https://docs.python.org/3/library/asyncio-dev.html#asyncio-debug-mode) to get the traceback where the task was created:
```
asyncio.run(main(), debug=True)

```

Output in debug mode:
```
Task exception was never retrieved
future: <Task finished coro=<bug() done, defined at test.py:3>
  exception=Exception('not consumed') created at asyncio/tasks.py:321>
source_traceback: Object created at (most recent call last):
 File "../t.py", line 9, in <module>
  asyncio.run(main(), debug=True)
< .. >
Traceback (most recent call last):
 File "../t.py", line 4, in bug
  raise Exception("not consumed")
Exception: not consumed

```

Title: Method Objects in Python C API
URL: https://docs.python.org/3/c-api/method.html
Summary: This section explains PyInstanceMethod_GET_FUNCTION, a macro that retrieves the function from a PyInstanceMethod, returning a borrowed reference. It also discusses method objects, noting that they are bound to instances of user-defined classes and that unbound methods associated with class objects are no longer available.

    _Return value: Borrowed reference._
Macro version of [`PyInstanceMethod_Function()`](https://docs.python.org/3/c-api/method.html#c.PyInstanceMethod_Function "PyInstanceMethod_Function") which avoids error checking.
# Method Objects[¶](https://docs.python.org/3/c-api/method.html#method-objects "Link to this heading")
Methods are bound function objects. Methods are always bound to an instance of a user-defined class. Unbound methods (methods bound to a class object) are no longer available. 

[PyTypeObject](https://docs.python.org/3/c-api/type.html#c.PyTypeObject "PyTypeObject")PyMethod_Type[¶](https://docs.python.org/3/c-api/method.html#c.PyMethod_Type "Link to this definition")
    
This instance of [`PyTypeObject`](https://docs.python.org/3/c-api/type.html#c.PyTypeObject "PyTypeObject") represents the Python method type. This is exposed to Python programs as `types.MethodType`. 

intPyMethod_Check([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*o)[¶](https://docs.python.org/3/c-api/method.html#c.PyMethod_Check "Link to this definition")
    
Return true if _o_ is a method object (has type [`PyMethod_Type`](https://docs.python.org/3/c-api/method.html#c.PyMethod_Type "PyMethod_Type")). The parameter must not be `NULL`. This function always succeeds. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyMethod_New([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*func, [PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*self)[¶](https://docs.python.org/3/c-api/method.html#c.PyMethod_New "Link to this definition")
    _Return value: New reference._
Return a new method object, with _func_ being any callable object and _self_ the instance the method should be bound. _func_ is the function that will be called when the method is called. _self_ must not be `NULL`. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyMethod_Function([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*meth)[¶](https://docs.python.org/3/c-api/method.html#c.PyMethod_Function "Link to this definition")
    _Return value: Borrowed reference._
Return the function object associated with the method _meth_. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyMethod_GET_FUNCTION([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*meth)[¶](https://docs.python.org/3/c-api/method.html#c.PyMethod_GET_FUNCTION "Link to this definition")
    _Return value: Borrowed reference._
Macro version of [`PyMethod_Function()`](https://docs.python.org/3/c-api/method.html#c.PyMethod_Function "PyMethod_Function") which avoids error checking. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyMethod_Self([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*meth)[¶](https://docs.python.org/3/c-api/method.html#c.PyMethod_Self "Link to this definition")
    _Return value: Borrowed reference._
Return the instance associated with the method _meth_. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyMethod_GET_SELF([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*meth)[¶](https://docs.python.org/3/c-api/method.html#c.PyMethod_GET_SELF "Link to this definition")

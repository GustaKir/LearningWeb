Title: Documentation Updates in Python
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This section outlines recent documentation changes, including the addition of the `attr` parameter to the `curses.window.vline()` function and corrections regarding the expected types of parameters in `compileall.compile_dir()` and `compileall.compile_file()`.
---

### Documentation[¶](https://docs.python.org/3/whatsnew/changelog.html#id179 "Link to this heading")
  * [gh-100616](https://github.com/python/cpython/issues/100616): Document existing `attr` parameter to [`curses.window.vline()`](https://docs.python.org/3/library/curses.html#curses.window.vline "curses.window.vline") function in [`curses`](https://docs.python.org/3/library/curses.html#module-curses "curses: An interface to the curses library, providing portable terminal handling. \(Unix\)").
  * [gh-100472](https://github.com/python/cpython/issues/100472): Remove claim in documentation that the `stripdir`, `prependdir` and `limit_sl_dest` parameters of [`compileall.compile_dir()`](https://docs.python.org/3/library/compileall.html#compileall.compile_dir "compileall.compile_dir") and [`compileall.compile_file()`](https://docs.python.org/3/library/compileall.html#compileall.compile_file "compileall.compile_file") could be [`bytes`](https://docs.python.org/3/library/stdtypes.html#bytes "bytes").
  * [bpo-25377](https://bugs.python.org/issue?@action=redirect&bpo=25377): Clarify use of octal format of mode argument in help(os.chmod) as well as help(os.fchmod)


### Tests[¶](https://docs.python.org/3/whatsnew/changelog.html#id180 "Link to this heading")
  * [gh-100454](https://github.com/python/cpython/issues/100454): Start running SSL tests with OpenSSL 3.1.0-beta1.
  * [gh-100086](https://github.com/python/cpython/issues/100086): The Python test runner (libregrtest) now logs Python build information like “debug” vs “release” build, or LTO and PGO optimizations. Patch by Victor Stinner.
  * [gh-93018](https://github.com/python/cpython/issues/93018): Make two tests forgiving towards host system libexpat with backported security fixes applied.


### Build[¶](https://docs.python.org/3/whatsnew/changelog.html#id181 "Link to this heading")
  * [gh-100540](https://github.com/python/cpython/issues/100540): Removed the `--with-system-ffi` `configure` option; `libffi` must now always be supplied by the system on all non-Windows platforms. The option has had no effect on non-Darwin platforms for several releases, and in 3.11 only had the non-obvious effect of invoking `pkg-config` to find `libffi` and never setting `-DUSING_APPLE_OS_LIBFFI`. Now on Darwin platforms `configure` will first check for the OS `libffi` and then fall back to the same processing as other platforms if it is not found.
  * [gh-88267](https://github.com/python/cpython/issues/88267): Avoid exporting Python symbols in linked Windows applications when the core is built as static.
  * [bpo-41916](https://bugs.python.org/issue?@action=redirect&bpo=41916): Allow override of ac_cv_cxx_thread so that cross compiled python can set -pthread for CXX.


### Windows[¶](https://docs.python.org/3/whatsnew/changelog.html#id182 "Link to this heading")
  * [gh-100180](https://github.com/python/cpython/issues/100180): Update Windows installer to OpenSSL 1.1.1s
  * [gh-99191](https://github.com/python/cpython/issues/99191): Use `_MSVC_LANG >= 202002L` instead of less-precise `_MSC_VER >=1929` to more accurately test for C++20 support in `PC/_wmimodule.cpp`.
  * [gh-79218](https://github.com/python/cpython/issues/79218): Define `MS_WIN64` for Mingw-w64 64bit, fix cython compilation failure.
  * [gh-99941](https://github.com/python/cpython/issues/99941): Ensure that [`asyncio.Protocol.data_received()`](https://docs.python.org/3/library/asyncio-protocol.html#asyncio.Protocol.data_received "asyncio.Protocol.data_received") receives an immutable [`bytes`](https://docs.python.org/3/library/stdtypes.html#bytes "bytes") object (as documented), instead of [`bytearray`](https://docs.python.org/3/library/stdtypes.html#bytearray "bytearray").
  * [bpo-43984](https://bugs.python.org/issue?@action=redirect&bpo=43984): [`winreg.SetValueEx()`](https://docs.python.org/3/library/winreg.html#winreg.SetValueEx "winreg.SetValueEx") now leaves the target value untouched in the case of conversion errors. Previously, `-1` would be written in case of such errors.
  * [bpo-34816](https://bugs.python.org/issue?@action=redirect&bpo=34816): `hasattr(ctypes.windll, 'nonexistant')` now returns `False` instead of raising [`OSError`](https://docs.python.org/3/library/exceptions.html#OSError "OSError").


### macOS[¶](https://docs.python.org/3/whatsnew/changelog.html#id183 "Link to this heading")
  * [gh-100180](https://github.com/python/cpython/issues/100180): Update macOS installer to OpenSSL 1.1.1s
  * [gh-100540](https://github.com/python/cpython/issues/100540): Removed obsolete `dlfcn.h` shim from the `_ctypes` extension module, which has not been necessary since Mac OS X 10.2.

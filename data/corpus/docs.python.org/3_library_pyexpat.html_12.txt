Title: XML Error: Amplification Limit Breach
URL: https://docs.python.org/3/library/pyexpat.html
Summary: This section describes the XML_ERROR_AMPLIFICATION_LIMIT_BREACH, which indicates that the limit on input amplification factor has been breached in XML parsing. It also notes that the encoding string in XML output must comply with established standards, highlighting valid and invalid examples.
---

xml.parsers.expat.errors.XML_ERROR_AMPLIFICATION_LIMIT_BREACH[¶](https://docs.python.org/3/library/pyexpat.html#xml.parsers.expat.errors.XML_ERROR_AMPLIFICATION_LIMIT_BREACH "Link to this definition")
    
The limit on input amplification factor (from DTD and entities) has been breached.
Footnotes
[[1](https://docs.python.org/3/library/pyexpat.html#id1)]
The encoding string included in XML output should conform to the appropriate standards. For example, “UTF-8” is valid, but “UTF8” is not. See <https://www.w3.org/TR/2006/REC-xml11-20060816/#NT-EncodingDecl> and <https://www.iana.org/assignments/character-sets/character-sets.xhtml>.

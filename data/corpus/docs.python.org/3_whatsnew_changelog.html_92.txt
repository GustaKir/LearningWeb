Title: Python Changelog Updates
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This section details recent patches including updates to HTTPSConnection for IPv6 proxy handling, fixes for misfolding issues in email address lists, adjustments to BufferedReader and BufferedRandom methods to prevent negative offsets, and a traceback resolution in argparse when suppressing mutual options.
---

Patched by Furkan Onder and Anilyka Barry.
  * [gh-100985](https://github.com/python/cpython/issues/100985): Update HTTPSConnection to consistently wrap IPv6 Addresses when using a proxy.
  * [gh-100884](https://github.com/python/cpython/issues/100884): email: fix misfolding of comma in address-lists over multiple lines in combination with unicode encoding.
  * [gh-95782](https://github.com/python/cpython/issues/95782): Fix `io.BufferedReader.tell()`, `io.BufferedReader.seek()`, `_pyio.BufferedReader.tell()`, `io.BufferedRandom.tell()`, `io.BufferedRandom.seek()` and `_pyio.BufferedRandom.tell()` being able to return negative offsets.
  * [gh-96310](https://github.com/python/cpython/issues/96310): Fix a traceback in [`argparse`](https://docs.python.org/3/library/argparse.html#module-argparse "argparse: Command-line option and argument parsing library.") when all options in a mutually exclusive group are suppressed.
  * [gh-93205](https://github.com/python/cpython/issues/93205): Fixed a bug in [`logging.handlers.TimedRotatingFileHandler`](https://docs.python.org/3/library/logging.handlers.html#logging.handlers.TimedRotatingFileHandler "logging.handlers.TimedRotatingFileHandler") where multiple rotating handler instances pointing to files with the same name but different extensions would conflict and not delete the correct files.
  * [bpo-31116](https://bugs.python.org/issue?@action=redirect&bpo=31116): Add Z85 encoding to `base64`.
  * [bpo-44865](https://bugs.python.org/issue?@action=redirect&bpo=44865): Add missing call to localization function in [`argparse`](https://docs.python.org/3/library/argparse.html#module-argparse "argparse: Command-line option and argument parsing library.").
  * [bpo-43952](https://bugs.python.org/issue?@action=redirect&bpo=43952): Fix [`multiprocessing.connection.Listener.accept()`](https://docs.python.org/3/library/multiprocessing.html#multiprocessing.connection.Listener.accept "multiprocessing.connection.Listener.accept") to accept empty bytes as authkey. Not accepting empty bytes as key causes it to hang indefinitely.
  * [bpo-42125](https://bugs.python.org/issue?@action=redirect&bpo=42125): linecache: get module name from `__spec__` if available. This allows getting source code for the `__main__` module when a custom loader is used.
  * [bpo-41122](https://bugs.python.org/issue?@action=redirect&bpo=41122): Failing to pass arguments properly to [`functools.singledispatchmethod()`](https://docs.python.org/3/library/functools.html#functools.singledispatchmethod "functools.singledispatchmethod") now throws a TypeError instead of hitting an index out of bounds internally.
  * [bpo-40818](https://bugs.python.org/issue?@action=redirect&bpo=40818): The asyncio REPL now runs [`sys.__interactivehook__`](https://docs.python.org/3/library/sys.html#sys.__interactivehook__ "sys.__interactivehook__") on startup. The default implementation of [`sys.__interactivehook__`](https://docs.python.org/3/library/sys.html#sys.__interactivehook__ "sys.__interactivehook__") provides auto-completion to the asyncio REPL. Patch contributed by Rémi Lapeyre.
  * [bpo-33775](https://bugs.python.org/issue?@action=redirect&bpo=33775): Add ‘default’ and ‘version’ help text for localization in argparse.


### Documentation[¶](https://docs.python.org/3/whatsnew/changelog.html#id92 "Link to this heading")

  * [gh-109653](https://github.com/python/cpython/issues/109653): Improve import time of [`uuid`](https://docs.python.org/3/library/uuid.html#module-uuid "uuid: UUID objects \(universally unique identifiers\) according to RFC 4122") on Linux.

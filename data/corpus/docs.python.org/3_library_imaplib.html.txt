# `imaplib` — IMAP4 protocol client[¶](https://docs.python.org/3/library/imaplib.html#module-imaplib "Link to this heading")
**Source code:** [Lib/imaplib.py](https://github.com/python/cpython/tree/3.13/Lib/imaplib.py)
This module defines three classes, [`IMAP4`](https://docs.python.org/3/library/imaplib.html#imaplib.IMAP4 "imaplib.IMAP4"), [`IMAP4_SSL`](https://docs.python.org/3/library/imaplib.html#imaplib.IMAP4_SSL "imaplib.IMAP4_SSL") and [`IMAP4_stream`](https://docs.python.org/3/library/imaplib.html#imaplib.IMAP4_stream "imaplib.IMAP4_stream"), which encapsulate a connection to an IMAP4 server and implement a large subset of the IMAP4rev1 client protocol as defined in [**RFC 2060**](https://datatracker.ietf.org/doc/html/rfc2060.html). It is backward compatible with IMAP4 ([**RFC 1730**](https://datatracker.ietf.org/doc/html/rfc1730.html)) servers, but note that the `STATUS` command is not supported in IMAP4.
[Availability](https://docs.python.org/3/library/intro.html#availability): not WASI.
This module does not work or is not available on WebAssembly. See [WebAssembly platforms](https://docs.python.org/3/library/intro.html#wasm-availability) for more information.
Three classes are provided by the [`imaplib`](https://docs.python.org/3/library/imaplib.html#module-imaplib "imaplib: IMAP4 protocol client \(requires sockets\).") module, [`IMAP4`](https://docs.python.org/3/library/imaplib.html#imaplib.IMAP4 "imaplib.IMAP4") is the base class: 

_class_ imaplib.IMAP4(_host =''_, _port =IMAP4_PORT_, _timeout =None_)[¶](https://docs.python.org/3/library/imaplib.html#imaplib.IMAP4 "Link to this definition")
    
This class implements the actual IMAP4 protocol. The connection is created and protocol version (IMAP4 or IMAP4rev1) is determined when the instance is initialized. If _host_ is not specified, `''` (the local host) is used. If _port_ is omitted, the standard IMAP4 port (143) is used. The optional _timeout_ parameter specifies a timeout in seconds for the connection attempt. If timeout is not given or is `None`, the global default socket timeout is used.
The [`IMAP4`](https://docs.python.org/3/library/imaplib.html#imaplib.IMAP4 "imaplib.IMAP4") class supports the [`with`](https://docs.python.org/3/reference/compound_stmts.html#with) statement. When used like this, the IMAP4 `LOGOUT` command is issued automatically when the `with` statement exits. E.g.:
>>>```
>>> fromimaplibimport IMAP4
>>> with IMAP4("domain.org") as M:
...   M.noop()
...
('OK', [b'Nothing Accomplished. d25if65hy903weo.87'])

Title: Python 3.11 Updates
URL: https://docs.python.org/3/whatsnew/3.11.html
Summary: This section outlines various updates in Python 3.11, including improvements to the `typing` library, which now avoids modifying external classes, and updates to the Unicode database, now at version 14.0.0. Additionally, the `unittest` module has introduced new methods `enterContext()` and `enterClassContext()`.

### unicodedata[¶](https://docs.python.org/3/whatsnew/3.11.html#unicodedata "Link to this heading")
  * The Unicode database has been updated to version 14.0.0. (Contributed by Benjamin Peterson in [bpo-45190](https://bugs.python.org/issue?@action=redirect&bpo=45190)).


### unittest[¶](https://docs.python.org/3/whatsnew/3.11.html#unittest "Link to this heading")
  * Added methods [`enterContext()`](https://docs.python.org/3/library/unittest.html#unittest.TestCase.enterContext "unittest.TestCase.enterContext") and [`enterClassContext()`](https://docs.python.org/3/library/unittest.html#unittest.TestCase.enterClassContext "unittest.TestCase.enterClassContext") of class [`TestCase`](https://docs.python.org/3/library/unittest.html#unittest.TestCase "unittest.TestCase"), method [`enterAsyncContext()`](https://docs.python.org/3/library/unittest.html#unittest.IsolatedAsyncioTestCase.enterAsyncContext "unittest.IsolatedAsyncioTestCase.enterAsyncContext") of class [`IsolatedAsyncioTestCase`](https://docs.python.org/3/library/unittest.html#unittest.IsolatedAsyncioTestCase "unittest.IsolatedAsyncioTestCase") and function [`unittest.enterModuleContext()`](https://docs.python.org/3/library/unittest.html#unittest.enterModuleContext "unittest.enterModuleContext"). (Contributed by Serhiy Storchaka in [bpo-45046](https://bugs.python.org/issue?@action=redirect&bpo=45046).)


### venv[¶](https://docs.python.org/3/whatsnew/3.11.html#venv "Link to this heading")
  * When new Python virtual environments are created, the _venv_ [sysconfig installation scheme](https://docs.python.org/3/library/sysconfig.html#installation-paths) is used to determine the paths inside the environment. When Python runs in a virtual environment, the same installation scheme is the default. That means that downstream distributors can change the default sysconfig install scheme without changing behavior of virtual environments. Third party code that also creates new virtual environments should do the same. (Contributed by Miro Hrončok in [bpo-45413](https://bugs.python.org/issue?@action=redirect&bpo=45413).)


### warnings[¶](https://docs.python.org/3/whatsnew/3.11.html#warnings "Link to this heading")
  * [`warnings.catch_warnings()`](https://docs.python.org/3/library/warnings.html#warnings.catch_warnings "warnings.catch_warnings") now accepts arguments for [`warnings.simplefilter()`](https://docs.python.org/3/library/warnings.html#warnings.simplefilter "warnings.simplefilter"), providing a more concise way to locally ignore warnings or convert them to errors. (Contributed by Zac Hatfield-Dodds in [bpo-47074](https://bugs.python.org/issue?@action=redirect&bpo=47074).)


### zipfile[¶](https://docs.python.org/3/whatsnew/3.11.html#zipfile "Link to this heading")
  * Added support for specifying member name encoding for reading metadata in a [`ZipFile`](https://docs.python.org/3/library/zipfile.html#zipfile.ZipFile "zipfile.ZipFile")’s directory and file headers. (Contributed by Stephen J. Turnbull and Serhiy Storchaka in [bpo-28080](https://bugs.python.org/issue?@action=redirect&bpo=28080).)
  * Added [`ZipFile.mkdir()`](https://docs.python.org/3/library/zipfile.html#zipfile.ZipFile.mkdir "zipfile.ZipFile.mkdir") for creating new directories inside ZIP archives. (Contributed by Sam Ezeh in [gh-49083](https://github.com/python/cpython/issues/49083).)
  * Added [`stem`](https://docs.python.org/3/library/zipfile.html#zipfile.Path.stem "zipfile.Path.stem"), [`suffix`](https://docs.python.org/3/library/zipfile.html#zipfile.Path.suffix "zipfile.Path.suffix") and [`suffixes`](https://docs.python.org/3/library/zipfile.html#zipfile.Path.suffixes "zipfile.Path.suffixes") to [`zipfile.Path`](https://docs.python.org/3/library/zipfile.html#zipfile.Path "zipfile.Path"). (Contributed by Miguel Brito in [gh-88261](https://github.com/python/cpython/issues/88261).)

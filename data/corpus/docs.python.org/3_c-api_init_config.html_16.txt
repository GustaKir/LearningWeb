Title: Python C API: Standard Input/Output Configuration
URL: https://docs.python.org/3/c-api/init_config.html
Summary: This section discusses the configuration of encoding and error handling for `sys.stdin`, `sys.stdout`, and `sys.stderr` in Python. It specifies that `sys.stderr` uses the 'backslashreplace' error handler and references the `PYTHONIOENCODING` environment variable for custom encoding settings. The default encoding is 'UTF-8' if `PyPreConfig.utf8_mode` is enabled, otherwise, it falls back to the locale encoding.
---

wchar_t*stdio_errors[¶](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.stdio_errors "Link to this definition")
    
Encoding and encoding errors of [`sys.stdin`](https://docs.python.org/3/library/sys.html#sys.stdin "sys.stdin"), [`sys.stdout`](https://docs.python.org/3/library/sys.html#sys.stdout "sys.stdout") and [`sys.stderr`](https://docs.python.org/3/library/sys.html#sys.stderr "sys.stderr") (but [`sys.stderr`](https://docs.python.org/3/library/sys.html#sys.stderr "sys.stderr") always uses `"backslashreplace"` error handler).
Use the [`PYTHONIOENCODING`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONIOENCODING) environment variable if it is non-empty.
Default encoding:
  * `"UTF-8"` if [`PyPreConfig.utf8_mode`](https://docs.python.org/3/c-api/init_config.html#c.PyPreConfig.utf8_mode "PyPreConfig.utf8_mode") is non-zero.
  * Otherwise, use the [locale encoding](https://docs.python.org/3/glossary.html#term-locale-encoding).


Default error handler:
  * On Windows: use `"surrogateescape"`.
  * `"surrogateescape"` if [`PyPreConfig.utf8_mode`](https://docs.python.org/3/c-api/init_config.html#c.PyPreConfig.utf8_mode "PyPreConfig.utf8_mode") is non-zero, or if the LC_CTYPE locale is “C” or “POSIX”.
  * `"strict"` otherwise.


See also [`PyConfig.legacy_windows_stdio`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.legacy_windows_stdio "PyConfig.legacy_windows_stdio"). 

inttracemalloc[¶](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.tracemalloc "Link to this definition")
    
Enable tracemalloc?
If non-zero, call [`tracemalloc.start()`](https://docs.python.org/3/library/tracemalloc.html#tracemalloc.start "tracemalloc.start") at startup.
Set by [`-X tracemalloc=N`](https://docs.python.org/3/using/cmdline.html#cmdoption-X) command line option and by the [`PYTHONTRACEMALLOC`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONTRACEMALLOC) environment variable.
Default: `-1` in Python mode, `0` in isolated mode. 

intperf_profiling[¶](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.perf_profiling "Link to this definition")
    
Enable compatibility mode with the perf profiler?
If non-zero, initialize the perf trampoline. See [Python support for the Linux perf profiler](https://docs.python.org/3/howto/perf_profiling.html#perf-profiling) for more information.
Set by [`-X perf`](https://docs.python.org/3/using/cmdline.html#cmdoption-X) command-line option and by the [`PYTHON_PERF_JIT_SUPPORT`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHON_PERF_JIT_SUPPORT) environment variable for perf support with stack pointers and [`-X perf_jit`](https://docs.python.org/3/using/cmdline.html#cmdoption-X) command-line option and by the [`PYTHON_PERF_JIT_SUPPORT`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHON_PERF_JIT_SUPPORT) environment variable for perf support with DWARF JIT information.
Default: `-1`.
Added in version 3.12. 

intuse_environment[¶](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.use_environment "Link to this definition")
    
Use [environment variables](https://docs.python.org/3/using/cmdline.html#using-on-envvars)?
If equals to zero, ignore the [environment variables](https://docs.python.org/3/using/cmdline.html#using-on-envvars).
Set to `0` by the [`-E`](https://docs.python.org/3/using/cmdline.html#cmdoption-E) environment variable.
Default: `1` in Python config and `0` in isolated config. 

intuser_site_directory[¶](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.user_site_directory "Link to this definition")
    
If non-zero, add the user site directory to [`sys.path`](https://docs.python.org/3/library/sys.html#sys.path "sys.path").
Set to `0` by the [`-s`](https://docs.python.org/3/using/cmdline.html#cmdoption-s) and [`-I`](https://docs.python.org/3/using/cmdline.html#cmdoption-I) command line options.
Set to `0` by the [`PYTHONNOUSERSITE`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONNOUSERSITE) environment variable.
Default: `1` in Python mode, `0` in isolated mode. 

intverbose[¶](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.verbose "Link to this definition")
    
Verbose mode. If greater than `0`, print a message each time a module is imported, showing the place (filename or built-in module) from which it is loaded.
If greater than or equal to `2`, print a message for each file that is checked for when searching for a module. Also provides information on module cleanup at exit.
Incremented by the [`-v`](https://docs.python.org/3/using/cmdline.html#cmdoption-v) command line option.
Set by the [`PYTHONVERBOSE`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONVERBOSE) environment variable value.
Default: `0`.

# List Objects[¶](https://docs.python.org/3/c-api/list.html#list-objects "Link to this heading") 

typePyListObject[¶](https://docs.python.org/3/c-api/list.html#c.PyListObject "Link to this definition")
    
This subtype of [`PyObject`](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject") represents a Python list object. 

[PyTypeObject](https://docs.python.org/3/c-api/type.html#c.PyTypeObject "PyTypeObject")PyList_Type[¶](https://docs.python.org/3/c-api/list.html#c.PyList_Type "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
This instance of [`PyTypeObject`](https://docs.python.org/3/c-api/type.html#c.PyTypeObject "PyTypeObject") represents the Python list type. This is the same object as [`list`](https://docs.python.org/3/library/stdtypes.html#list "list") in the Python layer. 

intPyList_Check([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*p)[¶](https://docs.python.org/3/c-api/list.html#c.PyList_Check "Link to this definition")
    
Return true if _p_ is a list object or an instance of a subtype of the list type. This function always succeeds. 

intPyList_CheckExact([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*p)[¶](https://docs.python.org/3/c-api/list.html#c.PyList_CheckExact "Link to this definition")
    
Return true if _p_ is a list object, but not an instance of a subtype of the list type. This function always succeeds. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyList_New([Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")len)[¶](https://docs.python.org/3/c-api/list.html#c.PyList_New "Link to this definition")
    _Return value: New reference.__Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Return a new list of length _len_ on success, or `NULL` on failure.
Note
If _len_ is greater than zero, the returned list object’s items are set to `NULL`. Thus you cannot use abstract API functions such as [`PySequence_SetItem()`](https://docs.python.org/3/c-api/sequence.html#c.PySequence_SetItem "PySequence_SetItem") or expose the object to Python code before setting all items to a real object with [`PyList_SetItem()`](https://docs.python.org/3/c-api/list.html#c.PyList_SetItem "PyList_SetItem") or [`PyList_SET_ITEM()`](https://docs.python.org/3/c-api/list.html#c.PyList_SET_ITEM "PyList_SET_ITEM"). The following APIs are safe APIs before the list is fully initialized: [`PyList_SetItem()`](https://docs.python.org/3/c-api/list.html#c.PyList_SetItem "PyList_SetItem") and [`PyList_SET_ITEM()`](https://docs.python.org/3/c-api/list.html#c.PyList_SET_ITEM "PyList_SET_ITEM"). 

[Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")PyList_Size([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*list)[¶](https://docs.python.org/3/c-api/list.html#c.PyList_Size "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Return the length of the list object in _list_ ; this is equivalent to `len(list)` on a list object.

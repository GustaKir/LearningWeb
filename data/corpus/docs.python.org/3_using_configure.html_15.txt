Title: Python Configure Environment Variables
URL: https://docs.python.org/3/using/configure.html
Summary: This document chunk outlines various environment variables used in the `./configure` script for compiling Python, including `CONFIGURE_CFLAGS_NODIST`, `BASECFLAGS`, `OPT`, `CFLAGS_ALIASING`, `CCSHARED`, and `CFLAGSFORSHARED`. It highlights their specific purposes, such as setting compiler flags and optimization options, with some variables introduced in particular Python versions.
---

CONFIGURE_CFLAGS_NODIST[¶](https://docs.python.org/3/using/configure.html#envvar-CONFIGURE_CFLAGS_NODIST "Link to this definition")
    
Value of [`CFLAGS_NODIST`](https://docs.python.org/3/using/configure.html#envvar-CFLAGS_NODIST) variable passed to the `./configure` script.
Added in version 3.5. 

BASECFLAGS[¶](https://docs.python.org/3/using/configure.html#envvar-BASECFLAGS "Link to this definition")
    
Base compiler flags. 

OPT[¶](https://docs.python.org/3/using/configure.html#envvar-OPT "Link to this definition")
    
Optimization flags. 

CFLAGS_ALIASING[¶](https://docs.python.org/3/using/configure.html#envvar-CFLAGS_ALIASING "Link to this definition")
    
Strict or non-strict aliasing flags used to compile `Python/dtoa.c`.
Added in version 3.7. 

CCSHARED[¶](https://docs.python.org/3/using/configure.html#envvar-CCSHARED "Link to this definition")
    
Compiler flags used to build a shared library.
For example, `-fPIC` is used on Linux and on BSD. 

CFLAGSFORSHARED[¶](https://docs.python.org/3/using/configure.html#envvar-CFLAGSFORSHARED "Link to this definition")
    
Extra C flags added for building the interpreter object files.
Default: `$(CCSHARED)` when [`--enable-shared`](https://docs.python.org/3/using/configure.html#cmdoption-enable-shared) is used, or an empty string otherwise. 

PY_CFLAGS[¶](https://docs.python.org/3/using/configure.html#envvar-PY_CFLAGS "Link to this definition")
    
Default: `$(BASECFLAGS) $(OPT) $(CONFIGURE_CFLAGS) $(CFLAGS) $(EXTRA_CFLAGS)`. 

PY_CFLAGS_NODIST[¶](https://docs.python.org/3/using/configure.html#envvar-PY_CFLAGS_NODIST "Link to this definition")
    
Default: `$(CONFIGURE_CFLAGS_NODIST) $(CFLAGS_NODIST) -I$(srcdir)/Include/internal`.
Added in version 3.5. 

PY_STDMODULE_CFLAGS[¶](https://docs.python.org/3/using/configure.html#envvar-PY_STDMODULE_CFLAGS "Link to this definition")
    
C flags used for building the interpreter object files.
Default: `$(PY_CFLAGS) $(PY_CFLAGS_NODIST) $(PY_CPPFLAGS) $(CFLAGSFORSHARED)`.
Added in version 3.7. 

PY_CORE_CFLAGS[¶](https://docs.python.org/3/using/configure.html#envvar-PY_CORE_CFLAGS "Link to this definition")
    
Default: `$(PY_STDMODULE_CFLAGS) -DPy_BUILD_CORE`.
Added in version 3.2. 

PY_BUILTIN_MODULE_CFLAGS[¶](https://docs.python.org/3/using/configure.html#envvar-PY_BUILTIN_MODULE_CFLAGS "Link to this definition")
    
Compiler flags to build a standard library extension module as a built-in module, like the [`posix`](https://docs.python.org/3/library/posix.html#module-posix "posix: The most common POSIX system calls \(normally used via module os\). \(Unix\)") module.
Default: `$(PY_STDMODULE_CFLAGS) -DPy_BUILD_CORE_BUILTIN`.
Added in version 3.8. 

PURIFY[¶](https://docs.python.org/3/using/configure.html#envvar-PURIFY "Link to this definition")
    
Purify command. Purify is a memory debugger program.
Default: empty string (not used).
### 3.5.3. Linker flags[¶](https://docs.python.org/3/using/configure.html#linker-flags "Link to this heading") 

LINKCC[¶](https://docs.python.org/3/using/configure.html#envvar-LINKCC "Link to this definition")
    
Linker command used to build programs like `python` and `_testembed`.
Default: `$(PURIFY) $(CC)`. 

CONFIGURE_LDFLAGS[¶](https://docs.python.org/3/using/configure.html#envvar-CONFIGURE_LDFLAGS "Link to this definition")
    
Value of [`LDFLAGS`](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS) variable passed to the `./configure` script.
Avoid assigning [`CFLAGS`](https://docs.python.org/3/using/configure.html#envvar-CFLAGS), [`LDFLAGS`](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS), etc. so users can use them on the command line to append to these values without stomping the pre-set values.
Added in version 3.2. 

LDFLAGS_NODIST[¶](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS_NODIST "Link to this definition")
    
[`LDFLAGS_NODIST`](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS_NODIST) is used in the same manner as [`CFLAGS_NODIST`](https://docs.python.org/3/using/configure.html#envvar-CFLAGS_NODIST). Use it when a linker flag should _not_ be part of [`LDFLAGS`](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS) once Python is installed ([gh-65320](https://github.com/python/cpython/issues/65320)).
In particular, [`LDFLAGS`](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS) should not contain:
  * the compiler flag `-L` (for setting the search path for libraries). The `-L` flags are processed from left to right, and any flags in [`LDFLAGS`](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS) would take precedence over user- and package-supplied `-L` flags.



CONFIGURE_LDFLAGS_NODIST[¶](https://docs.python.org/3/using/configure.html#envvar-CONFIGURE_LDFLAGS_NODIST "Link to this definition")
    
Value of [`LDFLAGS_NODIST`](https://docs.python.org/3/using/configure.html#envvar-LDFLAGS_NODIST) variable passed to the `./configure` script.
Added in version 3.8.

Title: PySys_SetArgv Function
URL: https://docs.python.org/3/c-api/init.html
Summary: Introduced in version 3.1.3 and deprecated in 3.11, the `PySys_SetArgv` function is maintained for backward compatibility. It is recommended to use `PyConfig.argv` and `PyConfig.parse_argv` instead. This function behaves similarly to `PySys_SetArgvEx()` with the `updatepath` set to `1`, unless the interpreter was started with the `-I` option.
---

```

Added in version 3.1.3.
Deprecated since version 3.11. 

voidPySys_SetArgv(intargc, wchar_t**argv)[¶](https://docs.python.org/3/c-api/init.html#c.PySys_SetArgv "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
This API is kept for backward compatibility: setting [`PyConfig.argv`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.argv "PyConfig.argv") and [`PyConfig.parse_argv`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.parse_argv "PyConfig.parse_argv") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
This function works like [`PySys_SetArgvEx()`](https://docs.python.org/3/c-api/init.html#c.PySys_SetArgvEx "PySys_SetArgvEx") with _updatepath_ set to `1` unless the **python** interpreter was started with the [`-I`](https://docs.python.org/3/using/cmdline.html#cmdoption-I).
Use [`Py_DecodeLocale()`](https://docs.python.org/3/c-api/sys.html#c.Py_DecodeLocale "Py_DecodeLocale") to decode a bytes string to get a wchar_t* string.
See also [`PyConfig.orig_argv`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.orig_argv "PyConfig.orig_argv") and [`PyConfig.argv`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.argv "PyConfig.argv") members of the [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Changed in version 3.4: The _updatepath_ value depends on [`-I`](https://docs.python.org/3/using/cmdline.html#cmdoption-I).
Deprecated since version 3.11. 

voidPy_SetPythonHome(constwchar_t*home)[¶](https://docs.python.org/3/c-api/init.html#c.Py_SetPythonHome "Link to this definition")
    _Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
This API is kept for backward compatibility: setting [`PyConfig.home`](https://docs.python.org/3/c-api/init_config.html#c.PyConfig.home "PyConfig.home") should be used instead, see [Python Initialization Configuration](https://docs.python.org/3/c-api/init_config.html#init-config).
Set the default “home” directory, that is, the location of the standard Python libraries. See [`PYTHONHOME`](https://docs.python.org/3/using/cmdline.html#envvar-PYTHONHOME) for the meaning of the argument string.
The argument should point to a zero-terminated character string in static storage whose contents will not change for the duration of the program’s execution. No code in the Python interpreter will change the contents of this storage.
Use [`Py_DecodeLocale()`](https://docs.python.org/3/c-api/sys.html#c.Py_DecodeLocale "Py_DecodeLocale") to decode a bytes string to get a wchar_t* string.
Deprecated since version 3.11.

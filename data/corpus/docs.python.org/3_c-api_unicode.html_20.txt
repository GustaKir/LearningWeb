Title: PyUnicode_EncodeCodePage Function
URL: https://docs.python.org/3/c-api/unicode.html
Summary: The `PyUnicode_EncodeCodePage` function encodes a Unicode object using a specified code page and returns a Python bytes object. If an exception occurs during encoding, it returns NULL. This function was introduced in Python version 3.3 and is part of the Stable ABI on Windows since version 3.7.
---

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyUnicode_EncodeCodePage(intcode_page, [PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*unicode, constchar*errors)[¶](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_EncodeCodePage "Link to this definition")
    _Return value: New reference.__Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable) on Windows since version 3.7._
Encode the Unicode object using the specified code page and return a Python bytes object. Return `NULL` if an exception was raised by the codec. Use `CP_ACP` code page to get the MBCS encoder.
Added in version 3.3.

## Methods and Slot Functions[¶](https://docs.python.org/3/c-api/unicode.html#methods-and-slot-functions "Link to this heading")
The following APIs are capable of handling Unicode objects and strings on input (we refer to them as strings in the descriptions) and return Unicode objects or integers as appropriate.
They all return `NULL` or `-1` if an exception occurs. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyUnicode_Concat([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*left, [PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*right)[¶](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_Concat "Link to this definition")
    _Return value: New reference.__Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Concat two strings giving a new Unicode string. 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyUnicode_Split([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*unicode, [PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*sep, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")maxsplit)[¶](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_Split "Link to this definition")
    _Return value: New reference.__Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Split a string giving a list of Unicode strings. If _sep_ is `NULL`, splitting will be done at all whitespace substrings. Otherwise, splits occur at the given separator. At most _maxsplit_ splits will be done. If negative, no limit is set. Separators are not included in the resulting list.
On error, return `NULL` with an exception set.
Equivalent to [`str.split()`](https://docs.python.org/3/library/stdtypes.html#str.split "str.split"). 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyUnicode_RSplit([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*unicode, [PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*sep, [Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")maxsplit)[¶](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_RSplit "Link to this definition")
    _Return value: New reference.__Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Similar to [`PyUnicode_Split()`](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_Split "PyUnicode_Split"), but splitting will be done beginning at the end of the string.
On error, return `NULL` with an exception set.
Equivalent to [`str.rsplit()`](https://docs.python.org/3/library/stdtypes.html#str.rsplit "str.rsplit"). 

[PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*PyUnicode_Splitlines([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*unicode, intkeepends)[¶](https://docs.python.org/3/c-api/unicode.html#c.PyUnicode_Splitlines "Link to this definition")
    _Return value: New reference.__Part of the[ Stable ABI](https://docs.python.org/3/c-api/stable.html#stable)._
Split a Unicode string at line breaks, returning a list of Unicode strings. CRLF is considered to be one line break. If _keepends_ is `0`, the Line break characters are not included in the resulting strings.

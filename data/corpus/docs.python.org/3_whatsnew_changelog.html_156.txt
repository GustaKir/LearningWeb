Title: Python 3 Changelog Updates
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This update includes patches addressing several issues: `unittest.mock.mock_open()` now properly calls the `close()` method on exit; improvements to `functools.singledispatchmethod` enhance its performance through caching; and adjustments to `unittest.mock` speccing improve its functionality.
---

Patch by Anthony Lee and Pieter Eendebak.
  * [bpo-44185](https://bugs.python.org/issue?@action=redirect&bpo=44185): [`unittest.mock.mock_open()`](https://docs.python.org/3/library/unittest.mock.html#unittest.mock.mock_open "unittest.mock.mock_open") will call the `close()` method of the file handle mock when it is exiting from the context manager. Patch by Samet Yaslan.
  * [bpo-40988](https://bugs.python.org/issue?@action=redirect&bpo=40988): Improve performance of [`functools.singledispatchmethod`](https://docs.python.org/3/library/functools.html#functools.singledispatchmethod "functools.singledispatchmethod") by caching the generated dispatch wrapper. Optimization suggested by frederico. Patch by @mental32, Alex Waygood and Pieter Eendebak.
  * [bpo-41768](https://bugs.python.org/issue?@action=redirect&bpo=41768): [`unittest.mock`](https://docs.python.org/3/library/unittest.mock.html#module-unittest.mock "unittest.mock: Mock object library.") speccing no longer calls class properties. Patch by Melanie Witt.
  * [bpo-18319](https://bugs.python.org/issue?@action=redirect&bpo=18319): Ensure `gettext(msg)` retrieve translations even if a plural form exists. In other words: `gettext(msg) == ngettext(msg, '', 1)`.
  * [bpo-17013](https://bugs.python.org/issue?@action=redirect&bpo=17013): Add `ThreadingMock` to [`unittest.mock`](https://docs.python.org/3/library/unittest.mock.html#module-unittest.mock "unittest.mock: Mock object library.") that can be used to create Mock objects that can wait until they are called. Patch by Karthikeyan Singaravelan and Mario Corchero.

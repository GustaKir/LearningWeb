Title: Library Updates in Python
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This section outlines recent updates in the Python standard library, including handling `ValueError` in `filecmp.dircmp`, improved error messages in `pickle`, and a fix for a segmentation fault in `asyncio.Task.get_coro`.
---

### Library[¶](https://docs.python.org/3/whatsnew/changelog.html#id50 "Link to this heading")
  * [gh-122400](https://github.com/python/cpython/issues/122400): Handle [`ValueError`](https://docs.python.org/3/library/exceptions.html#ValueError "ValueError")s raised by [`os.stat()`](https://docs.python.org/3/library/os.html#os.stat "os.stat") in [`filecmp.dircmp`](https://docs.python.org/3/library/filecmp.html#filecmp.dircmp "filecmp.dircmp") and [`filecmp.cmpfiles()`](https://docs.python.org/3/library/filecmp.html#filecmp.cmpfiles "filecmp.cmpfiles"). Patch by Bénédikt Tran.
  * [gh-122311](https://github.com/python/cpython/issues/122311): Fix some error messages in [`pickle`](https://docs.python.org/3/library/pickle.html#module-pickle "pickle: Convert Python objects to streams of bytes and back.").
  * [gh-122332](https://github.com/python/cpython/issues/122332): Fixed segfault with [`asyncio.Task.get_coro()`](https://docs.python.org/3/library/asyncio-task.html#asyncio.Task.get_coro "asyncio.Task.get_coro") when using an eager task factory.
  * [gh-105733](https://github.com/python/cpython/issues/105733): [`ctypes.ARRAY()`](https://docs.python.org/3/library/ctypes.html#ctypes.ARRAY "ctypes.ARRAY") is now [soft deprecated](https://docs.python.org/3/glossary.html#term-soft-deprecated): it no longer emits deprecation warnings and is not scheduled for removal.
  * [gh-122087](https://github.com/python/cpython/issues/122087): Restore [`inspect.ismethoddescriptor()`](https://docs.python.org/3/library/inspect.html#inspect.ismethoddescriptor "inspect.ismethoddescriptor") and [`inspect.isroutine()`](https://docs.python.org/3/library/inspect.html#inspect.isroutine "inspect.isroutine") returning `False` for [`functools.partial`](https://docs.python.org/3/library/functools.html#functools.partial "functools.partial") objects.
  * [gh-122170](https://github.com/python/cpython/issues/122170): Handle [`ValueError`](https://docs.python.org/3/library/exceptions.html#ValueError "ValueError")s raised by [`os.stat()`](https://docs.python.org/3/library/os.html#os.stat "os.stat") in [`linecache`](https://docs.python.org/3/library/linecache.html#module-linecache "linecache: Provides random access to individual lines from text files."). Patch by Bénédikt Tran.
  * [gh-82951](https://github.com/python/cpython/issues/82951): Serializing objects with complex `__qualname__` (such as unbound methods and nested classes) by name no longer involves serializing parent objects by value in pickle protocols < 4.
  * [gh-113785](https://github.com/python/cpython/issues/113785): [`csv`](https://docs.python.org/3/library/csv.html#module-csv "csv: Write and read tabular data to and from delimited files.") now correctly parses numeric fields (when used with [`csv.QUOTE_NONNUMERIC`](https://docs.python.org/3/library/csv.html#csv.QUOTE_NONNUMERIC "csv.QUOTE_NONNUMERIC") or [`csv.QUOTE_STRINGS`](https://docs.python.org/3/library/csv.html#csv.QUOTE_STRINGS "csv.QUOTE_STRINGS")) which start with an escape character.
  * [gh-122088](https://github.com/python/cpython/issues/122088): [`@warnings.deprecated`](https://docs.python.org/3/library/warnings.html#warnings.deprecated "warnings.deprecated") now copies the coroutine status of functions and methods so that [`inspect.iscoroutinefunction()`](https://docs.python.org/3/library/inspect.html#inspect.iscoroutinefunction "inspect.iscoroutinefunction") returns the correct result.
  * [gh-120930](https://github.com/python/cpython/issues/120930): Fixed a bug introduced by [gh-92081](https://github.com/python/cpython/issues/92081) that added an incorrect extra blank to encoded words occurring in wrapped headers.
  * [gh-121474](https://github.com/python/cpython/issues/121474): Fix missing sanity check for `parties` arg in [`threading.Barrier`](https://docs.python.org/3/library/threading.html#threading.Barrier "threading.Barrier") constructor. Patch by Clinton Christian (pygeek).
  * [gh-120289](https://github.com/python/cpython/issues/120289): Fixed the use-after-free issue in [`cProfile`](https://docs.python.org/3/library/profile.html#module-cProfile "cProfile") by disallowing `disable()` and `clear()` in external timers.


### IDLE[¶](https://docs.python.org/3/whatsnew/changelog.html#id51 "Link to this heading")
  * [gh-122482](https://github.com/python/cpython/issues/122482): Change About IDLE to direct users to discuss.python.org instead of the now unused idle-dev email and mailing list.

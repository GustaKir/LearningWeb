Title: Python 3 Changelog Highlights
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This entry discusses the updates in Python 3, including the raising of `ValueError` in `pathlib.PurePath.with_suffix()` for paths with an empty stem, the addition of `ssl.VERIFY_X509_PARTIAL_CHAIN` and `VERIFY_X509_STRICT` to the default SSL context, and allowing the creation of union types for `typing.Annotated`.
---

It now raises [`ValueError`](https://docs.python.org/3/library/exceptions.html#ValueError "ValueError"), just like [`pathlib.PurePath.with_suffix()`](https://docs.python.org/3/library/pathlib.html#pathlib.PurePath.with_suffix "pathlib.PurePath.with_suffix") does when called on a path with an empty stem, given a non-empty _suffix_ argument.
  * [gh-107361](https://github.com/python/cpython/issues/107361): Add [`ssl.VERIFY_X509_PARTIAL_CHAIN`](https://docs.python.org/3/library/ssl.html#ssl.VERIFY_X509_PARTIAL_CHAIN "ssl.VERIFY_X509_PARTIAL_CHAIN") and `VERIFY_X509_STRICT` to the default SSL context created with [`ssl.create_default_context()`](https://docs.python.org/3/library/ssl.html#ssl.create_default_context "ssl.create_default_context").
  * [gh-112281](https://github.com/python/cpython/issues/112281): Allow creating [union of types](https://docs.python.org/3/library/stdtypes.html#types-union) for [`typing.Annotated`](https://docs.python.org/3/library/typing.html#typing.Annotated "typing.Annotated") with unhashable metadata.
  * [gh-111775](https://github.com/python/cpython/issues/111775): Fix `importlib.resources.simple.ResourceHandle.open()` for text mode, added missed `stream` argument.
  * [gh-90095](https://github.com/python/cpython/issues/90095): Make .pdbrc and -c work with any valid pdb commands.
  * [gh-107625](https://github.com/python/cpython/issues/107625): Raise [`configparser.ParsingError`](https://docs.python.org/3/library/configparser.html#configparser.ParsingError "configparser.ParsingError") from [`read()`](https://docs.python.org/3/library/configparser.html#configparser.ConfigParser.read "configparser.ConfigParser.read") and [`read_file()`](https://docs.python.org/3/library/configparser.html#configparser.ConfigParser.read_file "configparser.ConfigParser.read_file") methods of [`configparser.ConfigParser`](https://docs.python.org/3/library/configparser.html#configparser.ConfigParser "configparser.ConfigParser") if a key without a corresponding value is continued (that is, followed by an indented line).
  * [gh-107155](https://github.com/python/cpython/issues/107155): Fix incorrect output of `help(x)` where `x` is a [`lambda`](https://docs.python.org/3/reference/expressions.html#lambda) function, which has an `__annotations__` dictionary attribute with a `"return"` key.
  * [gh-57141](https://github.com/python/cpython/issues/57141): Add option for _non-shallow_ comparisons to [`filecmp.dircmp`](https://docs.python.org/3/library/filecmp.html#filecmp.dircmp "filecmp.dircmp") like [`filecmp.cmp()`](https://docs.python.org/3/library/filecmp.html#filecmp.cmp "filecmp.cmp"). Original patch by Steven Ward. Enhanced by Tobias Rautenkranz
  * [gh-69990](https://github.com/python/cpython/issues/69990): `Profile.print_stats()` has been improved to accept multiple sort arguments. Patched by Chiu-Hsiang Hsu and Furkan Onder.
  * [gh-104061](https://github.com/python/cpython/issues/104061): Add `socket.SO_BINDTOIFINDEX` constant.
  * [gh-60346](https://github.com/python/cpython/issues/60346): Fix ArgumentParser inconsistent with parse_known_args.
  * [gh-102389](https://github.com/python/cpython/issues/102389): Add `windows_31j` to aliases for `cp932` codec
  * [gh-72249](https://github.com/python/cpython/issues/72249): `functools.partial`s of :func:`repr()` has been improved to include the [module](https://docs.python.org/3/glossary.html#term-module) name.

Title: Python 3 Changelog - Windows, Build, and C API Updates
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This section of the Python 3 changelog highlights updates related to Windows, including the py.exe launcher defaulting to Python 3 and removal of outdated icons. It also covers build improvements, such as fixes for cross-compiling pgen rules and updates to the OS X installer to work with OpenSSL 1.0.2h.
---

### Windows[¶](https://docs.python.org/3/whatsnew/changelog.html#id689 "Link to this heading")
  * [bpo-27064](https://bugs.python.org/issue?@action=redirect&bpo=27064): The py.exe launcher now defaults to Python 3. The Windows launcher `py.exe` no longer prefers an installed Python 2 version over Python 3 by default when used interactively.
  * [bpo-17500](https://bugs.python.org/issue?@action=redirect&bpo=17500): Remove unused and outdated icons. (See also: <https://github.com/python/pythondotorg/issues/945>)


### Build[¶](https://docs.python.org/3/whatsnew/changelog.html#id690 "Link to this heading")
  * [bpo-27229](https://bugs.python.org/issue?@action=redirect&bpo=27229): Fix the cross-compiling pgen rule for in-tree builds. Patch by Xavier de Gaye.
  * [bpo-26930](https://bugs.python.org/issue?@action=redirect&bpo=26930): Update OS X 10.5+ 32-bit-only installer to build and link with OpenSSL 1.0.2h.


### C API[¶](https://docs.python.org/3/whatsnew/changelog.html#id691 "Link to this heading")
  * [bpo-27186](https://bugs.python.org/issue?@action=redirect&bpo=27186): Add the PyOS_FSPath() function (part of [**PEP 519**](https://peps.python.org/pep-0519/)).
  * [bpo-26282](https://bugs.python.org/issue?@action=redirect&bpo=26282): PyArg_ParseTupleAndKeywords() now supports positional-only parameters.


### Tools/Demos[¶](https://docs.python.org/3/whatsnew/changelog.html#id692 "Link to this heading")
  * [bpo-26282](https://bugs.python.org/issue?@action=redirect&bpo=26282): Argument Clinic now supports positional-only and keyword parameters in the same function.

### Security[¶](https://docs.python.org/3/whatsnew/changelog.html#id693 "Link to this heading")
  * [bpo-26657](https://bugs.python.org/issue?@action=redirect&bpo=26657): Fix directory traversal vulnerability with http.server on Windows. This fixes a regression that was introduced in 3.3.4rc1 and 3.4.0rc1. Based on patch by Philipp Hagemeister.
  * [bpo-26313](https://bugs.python.org/issue?@action=redirect&bpo=26313): ssl.py _load_windows_store_certs fails if windows cert store is empty. Patch by Baji.
  * [bpo-25939](https://bugs.python.org/issue?@action=redirect&bpo=25939): On Windows open the cert store readonly in ssl.enum_certificates.

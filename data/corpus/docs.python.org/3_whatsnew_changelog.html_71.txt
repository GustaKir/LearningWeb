Title: Python Changelog Updates
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: This document includes various patch notes by Alex Waygood, summarizing updates such as the addition of a parameter check in `tarfile.TarFile.addfile()`, the removal of `DeprecationWarning` for specific parameter bindings, and a fix for a `TypeError` in `email.message.Message.get_payload()` when handling RFC 2231 encoded charsets.
---

Patch by Alex Waygood.
  * [gh-116931](https://github.com/python/cpython/issues/116931): Add parameter _fileobj_ check for [`tarfile.TarFile.addfile()`](https://docs.python.org/3/library/tarfile.html#tarfile.TarFile.addfile "tarfile.TarFile.addfile")
  * [gh-117995](https://github.com/python/cpython/issues/117995): Don’t raise [`DeprecationWarning`](https://docs.python.org/3/library/exceptions.html#DeprecationWarning "DeprecationWarning") when a [sequence](https://docs.python.org/3/glossary.html#term-sequence) of parameters is used to bind indexed, nameless placeholders. See also [gh-100668](https://github.com/python/cpython/issues/100668).
  * [gh-80361](https://github.com/python/cpython/issues/80361): Fix TypeError in [`email.message.Message.get_payload()`](https://docs.python.org/3/library/email.compat32-message.html#email.message.Message.get_payload "email.message.Message.get_payload") when the charset is [**RFC 2231**](https://datatracker.ietf.org/doc/html/rfc2231.html) encoded.
  * [gh-86650](https://github.com/python/cpython/issues/86650): Fix IndexError when parse some emails with invalid Message-ID (including one-off addresses generated by Microsoft Outlook).
  * [gh-117691](https://github.com/python/cpython/issues/117691): Improve the error messages emitted by [`tarfile`](https://docs.python.org/3/library/tarfile.html#module-tarfile "tarfile: Read and write tar-format archive files.") deprecation warnings relating to PEP 706. If a `filter` argument is not provided to `extract()` or `extractall`, the deprecation warning now points to the line in the user’s code where the relevant function was called. Patch by Alex Waygood.
  * [gh-115874](https://github.com/python/cpython/issues/115874): Fixed a possible segfault during garbage collection of `_asyncio.FutureIter` objects. Patch by Savannah Ostrowski.
  * [gh-115060](https://github.com/python/cpython/issues/115060): Speed up [`pathlib.Path.glob()`](https://docs.python.org/3/library/pathlib.html#pathlib.Path.glob "pathlib.Path.glob") by omitting an initial [`is_dir()`](https://docs.python.org/3/library/pathlib.html#pathlib.Path.is_dir "pathlib.Path.is_dir") call. As a result of this change, [`glob()`](https://docs.python.org/3/library/pathlib.html#pathlib.Path.glob "pathlib.Path.glob") can no longer raise [`OSError`](https://docs.python.org/3/library/exceptions.html#OSError "OSError").
  * [gh-77102](https://github.com/python/cpython/issues/77102): [`site`](https://docs.python.org/3/library/site.html#module-site "site: Module responsible for site-specific configuration.") module now parses `.pth` file with UTF-8 first, and [locale encoding](https://docs.python.org/3/glossary.html#term-locale-encoding) if `UnicodeDecodeError` happened. It supported only locale encoding before.
  * [gh-76785](https://github.com/python/cpython/issues/76785): We’ve exposed the low-level `_interpreters` module for the sake of the PyPI implementation of [**PEP 734**](https://peps.python.org/pep-0734/). It was sometimes available as the `_xxsubinterpreters` module and was formerly used only for testing. For the most part, it should be considered an internal module, like `_thread` and `_imp`.

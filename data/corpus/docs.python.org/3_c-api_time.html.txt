# PyTime C API[¶](https://docs.python.org/3/c-api/time.html#pytime-c-api "Link to this heading")
Added in version 3.13.
The clock C API provides access to system clocks. It is similar to the Python [`time`](https://docs.python.org/3/library/time.html#module-time "time: Time access and conversions.") module.
For C API related to the [`datetime`](https://docs.python.org/3/library/datetime.html#module-datetime "datetime: Basic date and time types.") module, see [DateTime Objects](https://docs.python.org/3/c-api/datetime.html#datetimeobjects).
## Types[¶](https://docs.python.org/3/c-api/time.html#types "Link to this heading") 

typePyTime_t[¶](https://docs.python.org/3/c-api/time.html#c.PyTime_t "Link to this definition")
    
A timestamp or duration in nanoseconds, represented as a signed 64-bit integer.
The reference point for timestamps depends on the clock used. For example, [`PyTime_Time()`](https://docs.python.org/3/c-api/time.html#c.PyTime_Time "PyTime_Time") returns timestamps relative to the UNIX epoch.
The supported range is around [-292.3 years; +292.3 years]. Using the Unix epoch (January 1st, 1970) as reference, the supported date range is around [1677-09-21; 2262-04-11]. The exact limits are exposed as constants: 

[PyTime_t](https://docs.python.org/3/c-api/time.html#c.PyTime_t "PyTime_t")PyTime_MIN[¶](https://docs.python.org/3/c-api/time.html#c.PyTime_MIN "Link to this definition")
    
Minimum value of [`PyTime_t`](https://docs.python.org/3/c-api/time.html#c.PyTime_t "PyTime_t"). 

[PyTime_t](https://docs.python.org/3/c-api/time.html#c.PyTime_t "PyTime_t")PyTime_MAX[¶](https://docs.python.org/3/c-api/time.html#c.PyTime_MAX "Link to this definition")
    
Maximum value of [`PyTime_t`](https://docs.python.org/3/c-api/time.html#c.PyTime_t "PyTime_t").
## Clock Functions[¶](https://docs.python.org/3/c-api/time.html#clock-functions "Link to this heading")
The following functions take a pointer to a [PyTime_t](https://docs.python.org/3/c-api/time.html#c.PyTime_t "PyTime_t") that they set to the value of a particular clock. Details of each clock are given in the documentation of the corresponding Python function.
The functions return `0` on success, or `-1` (with an exception set) on failure.
On integer overflow, they set the `PyExc_OverflowError` exception and set `*result` to the value clamped to the `[PyTime_MIN; PyTime_MAX]` range. (On current systems, integer overflows are likely caused by misconfigured system time.)
As any other C API (unless otherwise specified), the functions must be called with the [GIL](https://docs.python.org/3/glossary.html#term-GIL) held.

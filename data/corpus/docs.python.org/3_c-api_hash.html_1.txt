Title: Python C API Hash Functions
URL: https://docs.python.org/3/c-api/hash.html
Summary: This section describes functions from the Python C API for hashing, including `Py_HashPointer`, which hashes a pointer value without dereferencing it, and `PyObject_GenericHash`, a generic hashing function based on object identity, equivalent to `Py_HashPointer`.
---

[Py_hash_t](https://docs.python.org/3/c-api/hash.html#c.Py_hash_t "Py_hash_t")Py_HashPointer(constvoid*ptr)[¶](https://docs.python.org/3/c-api/hash.html#c.Py_HashPointer "Link to this definition")
    
Hash a pointer value: process the pointer value as an integer (cast it to `uintptr_t` internally). The pointer is not dereferenced.
The function cannot fail: it cannot return `-1`.
Added in version 3.13. 

[Py_hash_t](https://docs.python.org/3/c-api/hash.html#c.Py_hash_t "Py_hash_t")PyObject_GenericHash([PyObject](https://docs.python.org/3/c-api/structures.html#c.PyObject "PyObject")*obj)[¶](https://docs.python.org/3/c-api/hash.html#c.PyObject_GenericHash "Link to this definition")
    
Generic hashing function that is meant to be put into a type object’s `tp_hash` slot. Its result only depends on the object’s identity.
**CPython implementation detail:** In CPython, it is equivalent to [`Py_HashPointer()`](https://docs.python.org/3/c-api/hash.html#c.Py_HashPointer "Py_HashPointer").
Added in version 3.13.

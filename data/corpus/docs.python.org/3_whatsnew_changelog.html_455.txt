Title: Python 3.9.0 Alpha 1 Release Notes
URL: https://docs.python.org/3/whatsnew/changelog.html
Summary: The document outlines security updates in Python 3.9.0 Alpha 1, released on November 19, 2019. Notable changes include the use of `io.open_code()` in `runpy` for opening code files and the addition of new audit events for the `ctypes` module, along with a fix to an audit event for `os.system()`.
---

## Python 3.9.0 alpha 1[¶](https://docs.python.org/3/whatsnew/changelog.html#python-3-9-0-alpha-1 "Link to this heading")
_Release date: 2019-11-19_
### Security[¶](https://docs.python.org/3/whatsnew/changelog.html#id424 "Link to this heading")
  * [bpo-38722](https://bugs.python.org/issue?@action=redirect&bpo=38722): [`runpy`](https://docs.python.org/3/library/runpy.html#module-runpy "runpy: Locate and run Python modules without importing them first.") now uses [`io.open_code()`](https://docs.python.org/3/library/io.html#io.open_code "io.open_code") to open code files. Patch by Jason Killen.
  * [bpo-38622](https://bugs.python.org/issue?@action=redirect&bpo=38622): Add additional audit events for the [`ctypes`](https://docs.python.org/3/library/ctypes.html#module-ctypes "ctypes: A foreign function library for Python.") module.
  * [bpo-38418](https://bugs.python.org/issue?@action=redirect&bpo=38418): Fixes audit event for [`os.system()`](https://docs.python.org/3/library/os.html#os.system "os.system") to be named `os.system`.
  * [bpo-38243](https://bugs.python.org/issue?@action=redirect&bpo=38243): Escape the server title of [`xmlrpc.server.DocXMLRPCServer`](https://docs.python.org/3/library/xmlrpc.server.html#xmlrpc.server.DocXMLRPCServer "xmlrpc.server.DocXMLRPCServer") when rendering the document page as HTML. (Contributed by Donghee Na in [bpo-38243](https://bugs.python.org/issue?@action=redirect&bpo=38243).)
  * [bpo-38174](https://bugs.python.org/issue?@action=redirect&bpo=38174): Update vendorized expat library version to 2.2.8, which resolves [**CVE 2019-15903**](https://www.cve.org/CVERecord?id=CVE-2019-15903).
  * [bpo-37764](https://bugs.python.org/issue?@action=redirect&bpo=37764): Fixes email._header_value_parser.get_unstructured going into an infinite loop for a specific case in which the email header does not have trailing whitespace, and the case in which it contains an invalid encoded word. Patch by Ashwin Ramaswami.
  * [bpo-37461](https://bugs.python.org/issue?@action=redirect&bpo=37461): Fix an infinite loop when parsing specially crafted email headers. Patch by Abhilash Raj.
  * [bpo-37363](https://bugs.python.org/issue?@action=redirect&bpo=37363): Adds audit events for the range of supported run commands (see [Command line and environment](https://docs.python.org/3/using/cmdline.html#using-on-general)).
  * [bpo-37463](https://bugs.python.org/issue?@action=redirect&bpo=37463): ssl.match_hostname() no longer accepts IPv4 addresses with additional text after the address and only quad-dotted notation without trailing whitespaces. Some inet_aton() implementations ignore whitespace and all data after whitespace, e.g. ‘127.0.0.1 whatever’.
  * [bpo-37363](https://bugs.python.org/issue?@action=redirect&bpo=37363): Adds audit events for [`ensurepip`](https://docs.python.org/3/library/ensurepip.html#module-ensurepip "ensurepip: Bootstrapping the "pip" installer into an existing Python installation or virtual environment."), [`ftplib`](https://docs.python.org/3/library/ftplib.html#module-ftplib "ftplib: FTP protocol client \(requires sockets\)."), [`glob`](https://docs.python.org/3/library/glob.html#module-glob "glob: Unix shell style pathname pattern expansion."), [`imaplib`](https://docs.python.org/3/library/imaplib.html#module-imaplib "imaplib: IMAP4 protocol client \(requires sockets\)."), `nntplib`, [`pdb`](https://docs.python.org/3/library/pdb.html#module-pdb "pdb: The Python debugger for interactive interpreters."), [`poplib`](https://docs.python.org/3/library/poplib.html#module-poplib "poplib: POP3 protocol client \(requires sockets\)."), [`shutil`](https://docs.python.org/3/library/shutil.html#module-shutil "shutil: High-level file operations, including copying."), [`smtplib`](https://docs.python.org/3/library/smtplib.html#module-smtplib "smtplib: SMTP protocol client \(requires sockets\)."), [`sqlite3`](https://docs.python.org/3/library/sqlite3.html#module-sqlite3 "sqlite3: A DB-API 2.0 implementation using SQLite 3.x."), [`subprocess`](https://docs.python.org/3/library/subprocess.html#module-subprocess "subprocess: Subprocess management."), `telnetlib`, [`tempfile`](https://docs.python.org/3/library/tempfile.html#module-tempfile "tempfile: Generate temporary files and directories.") and [`os.listdir()`](https://docs.python.org/3/library/os.html#os.listdir "os.listdir"), [`os.scandir()`](https://docs.python.org/3/library/os.html#os.scandir "os.scandir") and [`breakpoint()`](https://docs.python.org/3/library/functions.html#breakpoint "breakpoint").

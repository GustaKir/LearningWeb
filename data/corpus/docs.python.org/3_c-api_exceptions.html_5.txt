Title: PyErr_SetExcFromWindowsErrWithFilenameObject Function
URL: https://docs.python.org/3/c-api/exceptions.html
Summary: The `PyErr_SetExcFromWindowsErrWithFilenameObject` function sets an exception based on a Windows error code, additionally accepting a filename parameter. If the filename is not `NULL`, it is used to set the `filename` attribute of an `OSError` exception. This function always returns NULL and is part of the Stable ABI on Windows since Python 3.7.
---

Similar to [`PyErr_SetExcFromWindowsErr()`](https://docs.python.org/3/c-api/exceptions.html#c.PyErr_SetExcFromWindowsErr "PyErr_SetExcFromWindowsErr"), with the additional behavior that if _filename_ is not `NULL`, it is passed to the constructor of [`OSError`](https://docs.python.org/3/library/exceptions.html#OSError "OSError") as a third parameter to be used to define the `filename` attribute of the exception instance.
[Availability](https://docs.python.org/3/library/intro.html#availability): Windows.

Added in version 3.3.

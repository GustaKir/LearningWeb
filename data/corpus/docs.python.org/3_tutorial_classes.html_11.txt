Title: Footnotes and Namespace Implementation in Python
URL: https://docs.python.org/3/tutorial/classes.html
Summary: Module objects in Python have a read-only attribute called `__dict__` that returns the dictionary for the module's namespace. This attribute should be used cautiously, as it exposes implementation details and violates the abstraction of namespace implementation.
---

```

Footnotes
[[1](https://docs.python.org/3/tutorial/classes.html#id1)]
Except for one thing. Module objects have a secret read-only attribute called [`__dict__`](https://docs.python.org/3/reference/datamodel.html#object.__dict__ "object.__dict__") which returns the dictionary used to implement the module’s namespace; the name `__dict__` is an attribute but not a global name. Obviously, using this violates the abstraction of namespace implementation, and should be restricted to things like post-mortem debuggers.

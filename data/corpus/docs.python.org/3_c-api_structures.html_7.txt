Title: PyMemberDef Structure Fields
URL: https://docs.python.org/3/c-api/structures.html
Summary: This section describes the fields of the PyMemberDef structure in the Python C API, including the member type, offset, flags, and documentation string. It provides references to related definitions and types used within the structure.
---

inttype[¶](https://docs.python.org/3/c-api/structures.html#c.PyMemberDef.type "Link to this definition")
    
The type of the member in the C struct. See [Member types](https://docs.python.org/3/c-api/structures.html#pymemberdef-types) for the possible values. 

[Py_ssize_t](https://docs.python.org/3/c-api/intro.html#c.Py_ssize_t "Py_ssize_t")offset[¶](https://docs.python.org/3/c-api/structures.html#c.PyMemberDef.offset "Link to this definition")
    
The offset in bytes that the member is located on the type’s object struct. 

intflags[¶](https://docs.python.org/3/c-api/structures.html#c.PyMemberDef.flags "Link to this definition")
    
Zero or more of the [Member flags](https://docs.python.org/3/c-api/structures.html#pymemberdef-flags), combined using bitwise OR. 

constchar*doc[¶](https://docs.python.org/3/c-api/structures.html#c.PyMemberDef.doc "Link to this definition")
    
The docstring, or NULL. The string should be static, no copy is made of it. Typically, it is defined using [`PyDoc_STR`](https://docs.python.org/3/c-api/intro.html#c.PyDoc_STR "PyDoc_STR").
By default (when [`flags`](https://docs.python.org/3/c-api/structures.html#c.PyMemberDef.flags "PyMemberDef.flags") is `0`), members allow both read and write access. Use the [`Py_READONLY`](https://docs.python.org/3/c-api/structures.html#c.Py_READONLY "Py_READONLY") flag for read-only access. Certain types, like [`Py_T_STRING`](https://docs.python.org/3/c-api/structures.html#c.Py_T_STRING "Py_T_STRING"), imply [`Py_READONLY`](https://docs.python.org/3/c-api/structures.html#c.Py_READONLY "Py_READONLY"). Only [`Py_T_OBJECT_EX`](https://docs.python.org/3/c-api/structures.html#c.Py_T_OBJECT_EX "Py_T_OBJECT_EX") (and legacy [`T_OBJECT`](https://docs.python.org/3/c-api/structures.html#c.T_OBJECT "T_OBJECT")) members can be deleted.
For heap-allocated types (created using [`PyType_FromSpec()`](https://docs.python.org/3/c-api/type.html#c.PyType_FromSpec "PyType_FromSpec") or similar), `PyMemberDef` may contain a definition for the special member `"__vectorcalloffset__"`, corresponding to [`tp_vectorcall_offset`](https://docs.python.org/3/c-api/typeobj.html#c.PyTypeObject.tp_vectorcall_offset "PyTypeObject.tp_vectorcall_offset") in type objects. These must be defined with `Py_T_PYSSIZET` and `Py_READONLY`, for example:
```
staticPyMemberDefspam_type_members[]={
{"__vectorcalloffset__",Py_T_PYSSIZET,
offsetof(Spam_object,vectorcall),Py_READONLY},
{NULL}/* Sentinel */
};

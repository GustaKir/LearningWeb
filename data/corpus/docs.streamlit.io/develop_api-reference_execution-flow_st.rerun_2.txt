Title: Using st.rerun in Streamlit
URL: https://docs.streamlit.io/develop/api-reference/execution-flow/st.rerun
Summary: This section discusses the usage of `st.rerun` in Streamlit applications, suggesting that callbacks might be a cleaner alternative and mentioning the utility of containers. It includes a simple example demonstrating how to update a header dynamically using `st.rerun` based on button interactions.
---

In many cases where `st.rerun` works, [callbacks](https://docs.streamlit.io/develop/api-reference/caching-and-state/st.session_state#use-callbacks-to-update-session-state) may be a cleaner alternative. [Containers](https://docs.streamlit.io/develop/api-reference/layout) may also be helpful.
### [](https://docs.streamlit.io/develop/api-reference/execution-flow/st.rerun#a-simple-example-in-three-variations)A simple example in three variations
###### [](https://docs.streamlit.io/develop/api-reference/execution-flow/st.rerun#using-strerun-to-update-an-earlier-header)Using `st.rerun` to update an earlier header
`import streamlit as st if "value" not in st.session_state:   st.session_state.value = "Title" ##### Option using st.rerun ##### st.header(st.session_state.value) if st.button("Foo"):   st.session_state.value = "Foo"   st.rerun() `
###### [](https://docs.streamlit.io/develop/api-reference/execution-flow/st.rerun#using-a-callback-to-update-an-earlier-header)Using a callback to update an earlier header
`##### Option using a callback ##### st.header(st.session_state.value) def update_value():   st.session_state.value = "Bar" st.button("Bar", on_click=update_value) `
###### [](https://docs.streamlit.io/develop/api-reference/execution-flow/st.rerun#using-containers-to-update-an-earlier-header)Using containers to update an earlier header
`##### Option using a container ##### container = st.container() if st.button("Baz"):   st.session_state.value = "Baz" container.header(st.session_state.value) `

Title: Button Design Anti-Patterns in Streamlit
URL: https://docs.streamlit.io/develop/concepts/design/buttons
Summary: This section discusses common mistakes in button design within Streamlit applications. It highlights issues such as nesting buttons inside each other and placing other widgets inside buttons, which can lead to unintended functionality and result execution failures.

## [](https://docs.streamlit.io/develop/concepts/design/buttons#anti-patterns)Anti-patterns
Here are some simplified examples of how buttons can go wrong. Be on the lookout for these common mistakes.
### [](https://docs.streamlit.io/develop/concepts/design/buttons#buttons-nested-inside-buttons)Buttons nested inside buttons
`import streamlit as st if st.button('Button 1'):   st.write('Button 1 was clicked') if st.button('Button 2'): # This will never be executed.     st.write('Button 2 was clicked') `
### [](https://docs.streamlit.io/develop/concepts/design/buttons#other-widgets-nested-inside-buttons)Other widgets nested inside buttons
`import streamlit as st if st.button('Sign up'):   name = st.text_input('Name') if name: # This will never be executed.     st.success(f'Welcome {name}') `
### [](https://docs.streamlit.io/develop/concepts/design/buttons#nesting-a-process-inside-a-button-without-saving-to-session-state)Nesting a process inside a button without saving to session state
`import streamlit as st import pandas as pd file = st.file_uploader("Upload a file", type="csv") if st.button('Get data'):   df = pd.read_csv(file) # This display will go away with the user's next action.   st.write(df) if st.button('Save'): # This will always error.   df.to_csv('data.csv') `
